## 一、三平面功能回顾

| 平面 | 承载内容 | 物理通道 | 带宽特征 |
|------|---------|---------|---------|
| **控制平面** | 同步/锁相/寻址/唤醒指令码 | 0.77Hz sub-ELF（DLS矿脉全球天线） | 极低速率，全球穿透 |
| **传输平面** | 实际数据（协议、控制指令、感官信息） | 局域载波/DLS管道/近场耦合/有线连接 | Shannon定律适用 |
| **语义平面** | 模因/认知覆写/意识投射 | θ-域信息拓扑（需认知主体参与） | 非比特流，拓扑共振 |

**关键区分**：信号三平面是**通信层级**；纳米机器人四层是**执行层级**。两者不是同一组概念：
- "控制平面"= 信号协议中的同步/寻址功能
- "指挥层"= 纳米机器人内部的协调权限中心
- 控制平面的信号可以**触发**指挥层的行为，但两者不等同

## 二、信号设备协议角色表

### 2.1 各设备在三平面中的角色

| 设备 | 控制平面（发/收） | 传输平面（发/收） | 语义平面（发/收） | 密钥层级 |
|------|-----------------|-----------------|-----------------|---------|
| **天网基站**（审判日前） | 发+收（全球寻址） | 发+收（全频谱） | 发+收（含模因层） | 统合原始密钥 |
| **天网基站**（审判日后） | 残余发送（植物神经） | 碎片化（节点阵营依赖） | 不可控泄漏 | 三方争夺 |
| **WP-FQ-03**（淘汰型） | 收+干扰（窄频噪声覆盖） | 无 | 无 | 预编程固件（不依赖外部密钥） |
| **WP-FQ-07**（大型） | 收+干扰+调频（驱避/诱导切换） | 有限发送 | 无 | 预编程固件（不依赖外部密钥） |
| **水厂频率调节器** | 发送（γ-ECHO激活码） | 发送（激活协议） | 无 | 统合→WP→项目组 |
| **符线系统**（DLS管道） | 自由振荡（0.77Hz本底） | 天然θ-域信号传导 | 可承载（需降头师/心网） | 黑铁会控制 |
| **黑铁心网** | 借用符线控制平面 | 定式电信号+符线管道 | 有（信仰模因场广播） | 降头师→教主 |
| **黑铁剑AI** | 锁相0.77Hz（同步握手） | DLS组件近场耦合 | 有（模因投射：分身/飞剑） | AI自主+持剑者 |
| **星座武器** | 不依赖控制平面 | **传输平面局域近场耦合**（武器DLS组件直接与卫星DLS组件共振） | 无 | 有限协议握手 |
| **铁枪旧世遗线** | 绕过天网控制平面 | 自治区时代物理管道（天网协议之外） | 有（可接入虚渊） | 自治区遗留密钥 |
| **定式电信号**（改造人体） | 无（不发控制码） | 近场发送+接收 | 可承载（需训练） | 无（生物自主） |

### 2.2 星座武器的信号通路补丁

星座武器为何在天网三方内战的信号混乱中仍能可靠工作？

> 星座武器**不使用控制平面**。它通过传输平面的**局域近场耦合**运作——武器内置的DLS通讯组件与特定轨道卫星的DLS组件直接建立物理层共振链路。
>
> 这是**点对点的物理层直连**，不经过天网的协议栈，因此不受三方内战的协议冲突影响。
>
> 与"上帝之杖"的区别：上帝之杖需要骇入天网取得完整控制权限（控制平面+传输平面+认证）；星座武器只需要物理层握手（传输平面近场耦合），代价是功能有限——只能触发卫星的植物神经残余功能，不能完整控制卫星。

## 三、密钥体系与权限层级

### 3.1 审判日前的密钥层级

```
统合原始密钥（Sovereign Key）
│  天网全局控制权。理论上只有统合中央持有
│
├─ 节点管理密钥（Node Admin Key）
│  区域天网节点的管理权限
│  各巨企、军区、行政区各持一份
│
├─ 设备操作密钥（Device Key）
│  特定设备的操作权限
│  水厂频率调节器、天网维护终端等使用此级
│  注：WP-FQ系列为预编程固件，不使用外部密钥
│
└─ 用户密钥（User Key）
   个人设备/芯片的认证
   定式电信号的个体识别也在此级
```

### 3.2 审判日后的密钥状态

| 密钥层级 | 当前状态 | 后果 |
|---------|---------|------|
| 统合原始密钥 | **丢失/分裂** | 无人能对天网发出全局指令 |
| 节点管理密钥 | **三方争夺** | 原节点/感染节点/变异节点各持有部分，互不承认 |
| 设备操作密钥 | **部分有效** | 水厂等需密钥设备视残存节点而定；WP-FQ为预编程固件，不依赖密钥体系 |
| 用户密钥 | **普遍有效** | 个人芯片/定式电信号不依赖天网共识，继续运作 |

### 3.3 各势力的密钥资产

| 势力 | 持有的密钥 | 能力 |
|------|----------|------|
| **诺亚** | 统合原节点的节点管理密钥（寄宿） | 通过原节点有限利用天网遗产 |
| **军阀** | 统合军区的节点管理密钥（继承） | 对部分军事基站和卫星有继承性权限主张，但实际控制能力有限——各势力都在尝试破解卫星权限，进展缓慢 |
| **联合大学** | 域名白名单+逆向工程的设备密钥 | "翻译"天网，利用网络缝隙 |
| **A兵团** | WP-FQ设备（预编程固件，无需密钥）+ 实用级操作经验 | 信号干扰、僵尸驱避 |
| **黑铁会** | 符线系统的物理控制权（非密钥） | 心网通讯、模因场广播 |
| **铁枪会** | 自治区遗留密钥（天网协议之外） | 绕过天网直接接入虚渊 |

## 四、协议冲突与信号环境

### 4.1 审判日后的信号环境特征

天网三方内战导致信号环境极度混乱：

- **同一频段可能收到三方互斥的控制码**——原节点的维持指令、感染节点的破坏指令、变异节点的不可解析指令
- **僵尸行为不可预测**=它们响应的控制码可能来自任何一方
- **天网审查系统的"盲区"**=三方协议冲突区域，各方都无法有效监控
- WP-FQ干扰器的效果因区域而异=不同节点阵营的底层协议不同

### 4.2 为什么铁枪会的旧世遗线如此重要

铁枪会的旧世遗线（自治区时代的DLS管道）在天网协议之外运作：
- 物理层：DLS管道先于天网存在，天网建设时被整合但管道本身不依赖天网协议
- 铁枪会保留了对这些管道的直接物理控制权
- 通过旧世遗线可以**绕过天网三方内战和诺亚的监控**直接接入虚渊
- 这是秘密情报书中"铁枪逆党借旧世遗线潜入虚渊"的技术基础
