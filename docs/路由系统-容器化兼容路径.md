# 路由系统 — 容器化兼容路径参考

本文从 [scripts/引擎/引擎_fs_兵器攻击路由.as](../scripts/引擎/引擎_fs_兵器攻击路由.as) / [scripts/引擎/引擎_fs_空手攻击路由.as](../scripts/引擎/引擎_fs_空手攻击路由.as) / [scripts/引擎/引擎_fs_战技路由.as](../scripts/引擎/引擎_fs_战技路由.as) 外移而来,这些是**渐进式容器化阶段**的兼容实现参考。

## 现状

`v3.0+` 的兵器/空手/战技容器化已经全量切换为"容器一定存在"路径,生产代码不再做符号存在性检测,也不维护旧帧路径作为回退。

这些兼容性实现保留在文档里是为了:
- 新增路由时若需要"渐进式容器化"(部分招式有容器、部分没有)可以照着改
- 排查"为什么容器化路径会卡住"时,对照旧帧路径理解期望行为

如果第二步 class 化时这些参考已无必要,可以直接删本文。

## 模式 A — 容器符号存在性检测(渐进式)

```as
_root.<XX>路由.__containerExistsCache = {};

_root.<XX>路由.检查容器符号存在 = function(unit:MovieClip, symbolName:String):Boolean {
    var cache:Object = _root.<XX>路由.__containerExistsCache;
    if (cache[symbolName] !== undefined) {
        return cache[symbolName];
    }
    var testInstanceName:String = "__containerExistTest_" + getTimer() + "_" + Math.floor(Math.random() * 10000);
    var testMan:MovieClip = unit.attachMovie(symbolName, testInstanceName, 9999);
    var exists:Boolean = (testMan != undefined);
    if (exists) {
        testMan.removeMovieClip();
    }
    cache[symbolName] = exists;
    return exists;
};
```

注意:`attachMovie` 探测会触发该符号的 `onClipEvent(load)`,渐进式阶段必须保证这些副作用在 9999 depth 上是良性的。

## 模式 B — 主角普攻连招开始(渐进式)

存在则走容器路径,不存在则回退旧帧:

```as
_root.<XX>攻击路由.主角普攻连招开始 = function(unit:MovieClip):Void {
    if (unit.兵种 !== "主角-男") {
        return;
    }
    var actionName:String = _root.<XX>攻击路由.获取普攻连招首帧标签(unit);
    unit.<XX>攻击名 = actionName;
    var symbolName:String = "<XX>攻击容器-" + actionName;
    var containerExists:Boolean = _root.<XX>攻击路由.检查容器符号存在(unit, symbolName);
    if (containerExists) {
        _root.发布消息("使用容器路径加载 " + symbolName);
        RoutingIntent.triggerStateTransitionJob(unit, "<XX>攻击", "容器", function(u:MovieClip):Void {
            _root.<XX>攻击路由.载入后跳转<XX>攻击容器(u.container, u);
        }, "容器");
        return;
    }
    _root.发布消息("容器不存在,使用旧帧路径加载 " + symbolName);
    RoutingIntent.createStateTransitionJob(unit, null, function(u:MovieClip):Void {
        _root.<XX>攻击路由.<XX>攻击帧载入(u.man, u);
    });
    RoutingIntent.submitStateTransitionJob(unit, "<XX>攻击", null);
};
```

## 模式 C — 旧帧路径的载入处理

容器符号缺失时,在旧 man.gotoAndPlay 之前需要补齐 onClipEvent(load) 的逻辑(升龙拳/上挑判定 + onUnload 写状态):

```as
_root.<XX>攻击路由.<XX>攻击帧载入 = function(man:MovieClip, unit:MovieClip):Void {
    if (unit._name == _root.控制目标) {
        unit.读取当前飞行状态();
        if (!unit.飞行浮空 && unit.被动技能.<升龙拳/上挑>.启用 && Key.isDown(unit.<触发键>)) {
            unit.跳横移速度 = unit.行走X速度;
            unit.跳跃中移动速度 = unit.行走X速度;
            unit.状态改变("<空手跳/兵器跳>");
            return;
        }
    }
    unit.格斗架势 = true;  // 空手才需要
    if (man != undefined) {
        RoutingIntent.bindContainerEndState(man, unit, "<XX>攻击结束");
    }
};
```

## 模式 D — 空手攻击标签跳转(渐进式)

```as
_root.空手攻击路由.空手攻击标签跳转 = function(unit:MovieClip, actionName:String):Void {
    unit.空手攻击名 = actionName;
    if (unit.兵种 !== "主角-男") {
        if (unit.man != undefined) {
            unit.man.gotoAndPlay(actionName);
        }
        return;
    }
    var symbolName:String = "空手攻击容器-" + actionName;
    var containerExists:Boolean = _root.空手攻击路由.检查容器符号存在(unit, symbolName);
    if (containerExists) {
        _root.发布消息("使用容器路径加载 " + symbolName);
        RoutingIntent.suppressOldManUnload(unit);
        RoutingIntent.triggerStateTransitionJob(unit, "空手攻击", "容器", function(u:MovieClip):Void {
            _root.空手攻击路由.载入后跳转空手攻击容器(u.container, u);
        }, "容器");
        return;
    }
    _root.发布消息("容器不存在,使用旧帧路径加载 " + symbolName);
    if (unit.man != undefined && !unit.man.__isDynamicMan) {
        _root.空手攻击路由.空手攻击帧载入(unit.man, unit);
        unit.man.gotoAndPlay(actionName);
        return;
    }
    RoutingIntent.createStateTransitionJob(unit, null, function(u:MovieClip):Void {
        _root.空手攻击路由.空手攻击帧载入(u.man, u);
        if (u.man != undefined) {
            u.man.gotoAndPlay(actionName);
        }
    });
    RoutingIntent.submitStateTransitionJob(unit, "空手攻击", null);
};
```

## 模式 E — 战技容器缺失回退

`v3.0+` 已统一假定战技容器存在。若需回退到旧帧:

```as
_root.战技路由.载入后跳转战技容器 = function(container:MovieClip, unit:MovieClip):Void {
    var 技能名:String = unit.技能名;
    var initObj:Object = RoutingLifecycle.buildPublicContainerInit(container);
    var newMan:MovieClip = unit.attachMovie("战技容器-" + 技能名, "man", 0, initObj);
    if (newMan == undefined) {
        _root.发布消息("战技容器-" + 技能名 + "符号缺失,尝试回退到旧跳帧逻辑");
        unit.gotoAndStop("战技");
        var fallbackMan:MovieClip = unit.man;
        if (fallbackMan != undefined) {
            RoutingLifecycle.bindMovement(fallbackMan);
            RoutingLifecycle.bindEndCleanup(fallbackMan, unit, undefined, "技能结束", "技能浮空");
            _root.战技路由.战技man载入后跳转_旧(fallbackMan, unit);
        }
        return;
    }
    _root.发布消息("战技容器-" + 技能名 + "加载完成,进入容器化战技逻辑");
    RoutingLifecycle.handleFloat(newMan, unit, "技能浮空");
    RoutingLifecycle.bindEndCleanup(newMan, unit, undefined, "技能结束", "技能浮空");
};
```

## 已知耦合

- 第二阶段 class 化入口是 [RoutingIntent.as](../scripts/类定义/org/flashNight/arki/unit/UnitComponent/Routing/RoutingIntent.as) 与 [RoutingLifecycle.as](../scripts/类定义/org/flashNight/arki/unit/UnitComponent/Routing/RoutingLifecycle.as)。`_root.路由基础` 暂时保留为旧 XML / 帧脚本兼容门面。
- `__stateTransitionJob` 字段约定见 [scripts/引擎/引擎_fs_路由基础.as](../scripts/引擎/引擎_fs_路由基础.as) "状态切换作业机制"注释块；实现归属是 `RoutingIntent`。
- 技能/战技/普攻容器的浮空、结束清理、动画完毕契约归属 `RoutingLifecycle`。
- 切换阶段必须屏蔽旧 man.onUnload(防止误写"普攻结束/<XX>攻击结束"覆盖新容器的结束逻辑) —— 现在统一走 `RoutingIntent.suppressOldManUnload(unit)`。
- 同帧跳转保护(`__skipWeaponChangeFrame` / `__skipBarehandChangeFrame`)由 `RoutingIntent.mark*SameFrameJump` / `RoutingIntent.is*SameFrameJump` 维护。
