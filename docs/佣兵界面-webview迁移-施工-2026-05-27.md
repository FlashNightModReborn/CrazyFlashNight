# 佣兵界面 WebView 迁移 — 施工记录 2026-05-27

## 目标

将 Flash 佣兵管理界面（`CRAZYFLASHER7MercenaryEmpire\LIBRARY\sprite\悬浮UI\Symbol 923.xml`）从 Flash 迁移为 HTML，通过 WebView2 面板系统被 C# 壳调用。复用战宠界面迁移已验证的 Panel 架构（panels.js + PanelHostController + Task 桥接）。

## 关键发现：Flash "佣兵报名"功能已废弃

Symbol 923 的 frame 0 脚本中，`佣兵报名()`、`佣兵赚取K点记录()`、`取消佣兵报名()` 三个联网函数全部被 `/* ... */` 注释包裹（line 71-243）。仅保留本地功能：
- `排列佣兵图标()` → 显示已雇佣佣兵列表
- `排列可雇用佣兵图标(页数)` → 分页显示可雇佣列表

**结论：只迁移"佣兵管理"和"查找佣兵"两个页面，不迁移"佣兵报名"。**

## 踩坑记录

### 1. hire_list / hire / dismiss 命令被 HandlePanelMessage switch 丢弃

- **现象**：切换到"雇佣兵"页面后 loading 卡死，返回列表页后出战/解雇按钮无反应，关闭后无法再次唤起
- **原因**：`WebOverlayForm.cs` 的 `HandlePanelMessage` switch 只覆盖了 `snapshot`、`deploy` 等已有命令，`hire_list`、`hire`、`dismiss` 三个 merc 专用命令不在任何 case 中，消息被静默丢弃。级联效应：JS 端 `_busy` 永久卡在 `true`，后续全部操作被 `if (_busy) return;` 拦截
- **修复**：将 3 个缺失命令加入 switch case 列表。这与战宠迁移 Issue #6 完全一致

### 2. handleEquipTooltip 委托 ArenaPanelService 导致回包 task 类型错误

- **现象**：hover 装备图标 tooltip 无显示
- **原因**：`MercPanelService.handleEquipTooltip` 直接委托 `ArenaPanelService.handleEquipTooltip`，后者发送响应 task 为 `arena_response`，但 `TaskRegistry` 注册的是 `merc_response`，回包无法被匹配
- **修复**：在 MercPanelService 中直接实现 tooltip 逻辑（复制 ArenaPanelService 代码但改用 `merc_response`），不再委托

### 3. handlePanelClose 调用 Flash UI 刷新导致关闭后 UI 阻塞

- **现象**：关闭佣兵面板后，鼠标无法与游戏 UI 交互（阻塞）
- **根因分析**：
  1. `ResolvePanelCloseGameCommand("mercs")` 返回 `"mercPanelClose"` → close 流程中 `TrySendGameCommand` 发了一条 socket 消息到 Flash
  2. 同时 `handlePanelClose` 调用 `_root.佣兵信息界面.排列佣兵图标()`，触发 Symbol 923 帧脚本 `gotoAndStop` + `attachMovie` 创建大量 MovieClip
  3. 这个 socket 通信 + Flash UI 操作干扰了 PanelHost 的 close 恢复序列（backdrop 移除/HUD 恢复/InputShield 清理）
- **修复**：
  - `handlePanelClose` 改为纯 no-op（WebView 不需要刷新 Flash UI）
  - `ResolvePanelCloseGameCommand("mercs")` 改为返回 `null`（参照 arena 模式：没有需要 AS2 清理的状态，close 时直接走 `_activePanel = null` + `PanelHost.ClosePanel()`）

## 2026-05-27 后续施工：装备显示迁移 + Bug 修复

### 装备显示迁移

将 arena 敌人卡片的装备显示体系复用到佣兵界面：

- **装备渲染**：文字标签 → `Icons.resolve()` 图标图片 + `onerror` 回退；11 槽固定遍历（含空槽虚线占位）
- **Tooltip**：自定义 `#merc-tooltip` → `PanelTooltip`（kshop 范式：即时基础 + 异步富文本 + 缓存）
- **CSS**：新增 `.merc-equip-cell`/`.merc-equip-empty`/`.merc-equip-level`/`.merc-equip-fallback`；删除 `.merc-tooltip` 系列样式

### Bug 修复: 装备图标不显示

- **现象**：装备槽位名和等级角标正常，但图标图片不显示
- **原因**：`merc-panel.js` 从未调用 `Icons.load()` 加载图标 manifest。其他面板（arena/kshop/intelligence）都在初始化时调用，佣兵面板遗漏。当 merc 是首个打开的面板时 `_map` 为 null，`Icons.resolve()` 始终返回 null
- **修复**：`createDOM()` 中 return 前调用 `Icons.load(function(){})`（与其他面板共享缓存）

### Bug 修复: 关闭面板后鼠标阻塞（续 Issue #3）

- **现象**：关闭佣兵面板后鼠标无法与游戏 UI 交互（Issue #3 修复后仍出现）
- **根因**：`requestClose()` 仅发送 Bridge 消息通知 C#，没有调用 `Panels.close()` 触发 `onClose`。C# `PanelHostController.DoClose()` 不会向 JS 回传 `panel_cmd.close`，因此 `onClose` 永不触发 → `_pendingReq`/`_busy`/`PanelTooltip` 等状态泄漏。对比 pet 面板：`requestClose()` 先调 `Panels.close()` 触发 `onClose` 清理，再通知 C#。此外 `handleHire` 在会话期间调用 `排列佣兵图标()` 触发 Flash 帧脚本，会干扰 close 恢复序列
- **修复**：
  1. `merc-panel.js` `requestClose()` 对齐 pet 模式：增加 `_busy` 守卫 + `Panels.close()` 调用（确保 onClose 触发 → PanelTooltip.hide() + _pendingReq/_busy 清理 + _active=null）
  2. `MercPanelService.as` `handleHire` 移除 `排列佣兵图标()` 调用（会话期间帧脚本会干扰 PanelHost close 恢复序列）
