### 性能分析报告：ActionScript 2（AS2）中的 `Object.prototype` 原型链注入

#### 1. 引言与目标

在 ActionScript 2 (AS2) 中，开发者可以通过修改 `Object.prototype` 为所有对象添加全局方法。然而，随着原型链上的方法数量增加，可能导致性能上的明显劣化。本次分析的目的是研究不同复杂度的原型链注入对 `Object` 常规操作（如创建、赋值、删除等）的性能影响，并量化这些影响以提供优化建议。

为了全面分析影响，我们设计了不同的测试场景，并按照轻度、中度和重度三种注入层次来评估原型链的性能代价。

#### 2. 测试设计

**测试场景**：
1. **对象创建测试**：衡量在不同注入级别下，创建新对象的时间开销。
2. **属性赋值与方法调用**：测试大量的属性赋值和方法调用操作，评估每次查找原型链时的性能消耗。
3. **属性删除测试**：分析删除对象属性时的性能表现。
4. **复杂操作测试**：模拟真实场景，包括属性赋值、删除和方法调用的组合，评估综合性能。

**注入复杂度**：
- **轻度注入**：注入 1-2 个方法到 `Object.prototype`。
- **中度注入**：注入 3-4 个方法到 `Object.prototype`。
- **重度注入**：注入 5 个及以上方法。

每个场景都与不进行任何注入的基准性能进行对比，重点分析不同注入层次下，原型链对常规操作带来的额外开销。

#### 3. 数据及分析

**测试结果**：

| 操作类型                | Baseline | Light Injection | Moderate Injection | Heavy Injection |
|-------------------------|----------|-----------------|--------------------|-----------------|
| **Object Creation**      | 42 ms    | 40 ms           | 41 ms              | 41 ms           |
| **Property Assignment**  | 117 ms   | 235 ms          | 358 ms             | 354 ms          |
| **Property Deletion**    | 43 ms    | 49 ms           | 42 ms              | 48 ms           |
| **Complex Operations**   | 50 ms    | 44 ms           | 79 ms              | 98 ms           |

**数据解释**：

1. **对象创建**：
   - **结果**：不论是轻度、中度还是重度注入，创建对象的时间都保持在 **40-42 ms** 之间。
   - **分析**：这是因为对象的创建主要依赖于内存分配，几乎不涉及原型链的查找。因此，原型链的复杂度不会对对象的创建产生显著影响。

2. **属性赋值与方法调用**：
   - **结果**：随着注入复杂度的增加，属性赋值的时间显著增加。轻度注入时，赋值时间从 **117 ms** 增加到 **235 ms**；中度注入时增加到 **358 ms**；重度注入的时间略有下降但仍保持在 **354 ms** 左右。
   - **分析**：在 AS2 中，每次赋值操作都会触发原型链查找。当 `Object.prototype` 上增加更多方法时，每次赋值操作需要遍历更长的原型链，导致性能劣化。在重度注入时，性能稍有稳定可能是因为查找层次达到一定上限，内存缓存或引擎优化起到了作用。

3. **属性删除**：
   - **结果**：无论注入复杂度如何，删除属性的开销相对稳定，在 **42-49 ms** 之间波动。
   - **分析**：属性删除主要针对对象自身的属性表进行操作，只有在属性不存在时才会触发原型链查找。因此，除非删除的属性在 `Object.prototype` 上，删除操作几乎不会受到原型链复杂度的影响。

4. **复杂操作**：
   - **结果**：随着注入复杂度的增加，复杂操作的时间逐步上升。从基线的 **50 ms** 增加到中度注入的 **79 ms**，最终在重度注入时达到 **98 ms**。
   - **分析**：复杂操作包括赋值、删除和方法调用，这些操作会频繁触发原型链查找。因此，注入的每个新方法都会对整体性能产生累积影响。特别是在重度注入时，原型链查找次数增加，导致整体性能劣化显著。

#### 4. 性能影响总结

- **原型链查找的影响**：原型链查找是性能下降的主要原因。每当注入新的方法时，`Object.prototype` 的长度增加，导致每次赋值或方法调用都需要遍历整个原型链。这种查找过程的开销在频繁的操作中逐渐累积，导致赋值和复杂操作的性能显著下降。
  
- **对象创建与属性删除的低影响**：对象创建和属性删除操作在不同注入层次下的性能表现几乎一致。这是因为这些操作大多不依赖于原型链上的方法，更多是与内存分配或对象自身属性表相关。

- **轻度、中度、重度注入的区别**：性能下降在轻度注入时较为温和，但从中度注入开始（超过 3 个方法），性能下降迅速加剧。重度注入的性能略有稳定，这可能是由于 AS2 引擎在处理极端情况时存在某些优化机制。

#### 5. 优化建议

- **避免全局原型链修改**：尽量避免直接修改全局的 `Object.prototype`。尤其是在大型项目或高频操作的场景下，原型链查找的性能开销可能会非常高。
  
- **局部化方法注入**：为特定对象或类注入方法，而不是修改 `Object.prototype`。通过局部化设计可以避免频繁查找原型链，提高性能。

- **减少原型链的深度**：避免在项目中设计过深的继承链，尤其是在继承链上存在大量公共方法的情况下。每次查找都会遍历整个链条，导致性能下降。

#### 6. 结论

通过测试，我们发现 AS2 中的 `Object.prototype` 注入对属性赋值和方法调用的性能影响最为显著。在轻度注入时，性能影响相对较小，但随着原型链中方法数量的增加，性能下降显著。因此，在性能敏感的场景中，应尽量避免对全局 `Object.prototype` 的频繁修改，使用更具局部性的设计策略以优化性能。



// 通用的时间测量工具
function timeMeasure(startTime, description) {
    var endTime = getTimer();
    trace(description + " - Time: " + (endTime - startTime) + " ms");
}

// 注入方法到 Object.prototype，根据注入的层次控制注入方法的数量
function injectPrototypeMethods(level) {
    if (level >= 1) {
        Object.prototype.newMethod1 = function() { return "Method 1"; };
    }
    if (level >= 2) {
        Object.prototype.newMethod2 = function() { return "Method 2"; };
        Object.prototype.newMethod3 = function() { return "Method 3"; };
    }
    if (level >= 3) {
        Object.prototype.newMethod4 = function() { return "Method 4"; };
        Object.prototype.newMethod5 = function() { return "Method 5"; };
        Object.prototype.newMethod6 = function() { return "Method 6"; };
        Object.prototype.newMethod7 = function() { return "Method 7"; };
        Object.prototype.newMethod8 = function() { return "Method 8"; };
    }
}

// 清理 Object.prototype 上的注入方法
function cleanupPrototypeMethods() {
    delete Object.prototype.newMethod1;
    delete Object.prototype.newMethod2;
    delete Object.prototype.newMethod3;
    delete Object.prototype.newMethod4;
    delete Object.prototype.newMethod5;
    delete Object.prototype.newMethod6;
    delete Object.prototype.newMethod7;
    delete Object.prototype.newMethod8;
}
// 测试对象创建的开销
function testObjectCreation(iterations, description) {
    var startTime = getTimer();
    for (var i = 0; i < iterations; i++) {
        var obj = {};   // 创建空对象
    }
    timeMeasure(startTime, description);
}

// 测试属性赋值和方法调用
function testPropertyAssignment(iterations, description) {
    var obj = {};
    var startTime = getTimer();
    
    for (var i = 0; i < iterations; i++) {
        obj['prop' + i] = i;               // 属性赋值
        if (obj.newMethod1) obj.newMethod1(); // 方法调用
        if (obj.newMethod2) obj.newMethod2(); // 方法调用
    }
    
    timeMeasure(startTime, description);
}

// 测试属性删除的开销
function testPropertyDeletion(iterations, description) {
    var obj = {};
    for (var i = 0; i < iterations; i++) {
        obj['prop' + i] = i;               // 先赋值
    }

    var startTime = getTimer();
    for (var i = 0; i < iterations; i++) {
        delete obj['prop' + i];            // 删除属性
    }
    timeMeasure(startTime, description);
}

// 测试复杂的操作，包括赋值、删除、调用等
function testComplexOperations(iterations, description) {
    var obj = {};
    var startTime = getTimer();
    
    for (var i = 0; i < iterations; i += 5) {
        obj['prop' + i] = i;               // 赋值
        obj['prop' + (i + 1)] = i + 1;
        obj['prop' + (i + 2)] = i + 2;
        delete obj['prop' + i];            // 删除部分属性
        if (obj.newMethod3) obj.newMethod3(); // 调用方法
        if (obj.newMethod4) obj.newMethod4(); // 调用方法
    }

    timeMeasure(startTime, description);
}
// 执行完整的测试序列，覆盖轻度、中度、重度注入场景
function runTests(iterations) {
    // 无原型链注入的基准测试
    testObjectCreation(iterations, "Baseline - Object Creation");
    testPropertyAssignment(iterations, "Baseline - Property Assignment");
    testPropertyDeletion(iterations, "Baseline - Property Deletion");
    testComplexOperations(iterations, "Baseline - Complex Operations");

    // 轻度注入
    injectPrototypeMethods(1);
    testObjectCreation(iterations, "Light Injection - Object Creation");
    testPropertyAssignment(iterations, "Light Injection - Property Assignment");
    testPropertyDeletion(iterations, "Light Injection - Property Deletion");
    testComplexOperations(iterations, "Light Injection - Complex Operations");

    // 清理注入的原型链方法
    cleanupPrototypeMethods();

    // 中度注入
    injectPrototypeMethods(2);
    testObjectCreation(iterations, "Moderate Injection - Object Creation");
    testPropertyAssignment(iterations, "Moderate Injection - Property Assignment");
    testPropertyDeletion(iterations, "Moderate Injection - Property Deletion");
    testComplexOperations(iterations, "Moderate Injection - Complex Operations");

    // 清理注入的原型链方法
    cleanupPrototypeMethods();

    // 重度注入
    injectPrototypeMethods(3);
    testObjectCreation(iterations, "Heavy Injection - Object Creation");
    testPropertyAssignment(iterations, "Heavy Injection - Property Assignment");
    testPropertyDeletion(iterations, "Heavy Injection - Property Deletion");
    testComplexOperations(iterations, "Heavy Injection - Complex Operations");

    // 清理注入的原型链方法
    cleanupPrototypeMethods();
}

// 开始测试
var iterations = 50000;  // 根据需求调整迭代次数
runTests(iterations);
