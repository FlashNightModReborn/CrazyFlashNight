## 3. 与设计的偏差（施工时发现并已处置）

1. **成就 id 号段 90001→7001xx-7006xx**：设计的「90001 起独立号段」实测撞车——`intelligence_tasks.json` 任务 id 正占用 90001-90011。改用 7001xx(基建)/7002xx(击杀)/7003xx(任务)/7004xx(技能)/7005xx(物品)/7006xx(经济)，全任务 id 区间（0-77/1万/2万/3万/4万/5万/6万/9万/12万/13万/15万/100万）无冲突。派生器保留任务 id 空间重叠告警。
2. **⚠AchievementMetrics.VALID 字面量→buildValid() 赋值式**：Flash CS6 的 **AS2 类编译器不接受对象字面量字符串键**（`{"中文键": true}` 报「需要标识符」；帧脚本里可以、类体里不行）。S2 编译 0 错是假阴性——当时类无任何引用根本没被编译，S5 帧脚本引用后才暴露（22 错）。改为 `public static var VALID:Object = buildValid();` + 函数体内 `v["键"] = true;` 赋值式；record() 首行加 `VALID == undefined` 惰性兜底（AS2 静态初始化次序防御）；derive 解析正则同步改为匹配 buildValid 函数体——**白名单单源约定不变，但格式锚点变了，改格式须两处联动**。
3. **hiddenReveals 增加 rewards 字段**：catalog hidden 条目 rewards 脱敏为 `[]` 后，已解锁 hidden 条目在 web 无处取奖励展示——hiddenReveals 从设计的 `{id,title,description}` 扩为含 `rewards:[{name,count}]`（AS2 raw JSON 本就有全文，按需回传范式不变）。同时 catalog hidden 条目的 objective 剔除 params（物品名/任务 id 线索），仅保留 {type,target}。
4. **商城两站 record 置于 `_root.强制存盘()` 之前**：计数随本次结账/领取同一笔 flush 原子落盘（设计只说成功分支，未指明与强制存盘的相对位置）。
5. **「宠物培养」口径 = `handleLevelUp`（灵石升级）**：设计锚 :821 实际落在 handleLevelUp 内（war-pet 灵石培养升级），非独立"培养"函数。
6. **harness 截图驱动补 `?tab=ach`**：便于后续视觉复核（`node tools/run-tasks-harness.js --shot=<png> --query="tab=ach"`）。
