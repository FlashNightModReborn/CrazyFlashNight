## 2. 引用拓扑：为什么"没有安全起点"

跨文档耦合**几乎不走 markdown 链接，而走 `稳定节名` 命名锚点**（这是好设计，见 §5）。但锚点引用构成的图**有环**，且高度集中：

### 2.1 引力井：08 是黑洞

```
被引用次数（按 `NN·` 锚点 + "见/对齐 NN" 统计）：
08 ← 00(21) 03(7) 05(6) 10(6) 18(12) 19(10) 13(3) README(5) + 01/02/06/07/12/14 各 1~2
```
几乎每篇都指向 08。08 = 事实上的中央枢纽，但**从未被显式声明为 hub**——所以写手不知道"哪些事实该归 08 管"。

### 2.2 紧耦合三角：08 ↔ 18 ↔ 19

```
18 → 19 : 10 次       19 → 18 : 8 次
18 → 08 : 12 次       19 → 08 : 10 次
```
18/19 互为"前传/政治线"，双双重压 08。三者**互引成环**。他们已用"主从约定 / 以 X 为准"压住了**逻辑**环（见 [18:5](../worldbuilding/18-天网起源：从天上的网到神谕之神.md#L5) 的"主从约定"），但**引用拓扑**仍是环——意味着没有任何一篇能脱离其他两篇被独立修改。

### 2.3 双向环：00 矩阵 ↔ 每一篇

00 登记每篇的 claim，每篇又回指 00 的归属分级。这是一个 hub-spoke 双向环，且 00 的"拟揭露候选"区是"当前主假说"区的**手工镜像**（见 §3.1）。

**结论**：环不在逻辑层（内容自洽），在**引用拓扑层**。压力来源 = 写手每次落笔要在脑里同时持有整张图的不变量。修复 = 把环拍成 **DAG**（§6 第二优先）。

## 4. 健康的承重护栏 —— 禁止在"拆护栏"时误伤

> ⚠ **给 GPT Pro / ultracode 的硬约束**：以下是廉价且承重的护栏，拆了会退回更糟的状态。整理时**只拆 §3 的冗余手工护栏，不准动以下任何一项**。

### 4.1 ✅ `稳定节名` 命名锚点 + 它的 validator —— 整个系统最值钱的设计
[tools/validate-doc-governance.js:255-300](../../tools/validate-doc-governance.js#L255-L300) 已实现一个轻量但有效的校验：
- 扫描所有含 `稳定节名` 的行，收集反引号内 `NN·...` 作为**定义**；
- 校验全库每个反引号 `NN·...` 引用都能解析到定义（防悬空引用）；
- 用 `STALE_WORLDBUILDING_ANCHORS` 正则**禁止**行号 / `#L` / `§N.` / `:line` 引用（防脆引用）。

**这是为什么 101 处跨节引用没有变成维护地狱的根本原因。** 必须保留，并**扩展**（让所有跨文档引用强制走它）。
> 注：该 validator **只扫描 `docs/worldbuilding/` + `worldbuilding-version-history.md`**（见 [validate-doc-governance.js:265](../../tools/validate-doc-governance.js#L265)）。本报告位于 `docs/reports/` 且文件名不同，**不在扫描范围**，不会触发治理门。

### 4.2 ✅ 三层 claim 分类法（共享框架 / 假说 / 支线真相）+ README 防火墙
概念正确——它防止提案被当硬 canon 落地。**问题在表达冗余（§3.1 的双登记），不在概念。** 保留概念，瘦身表达。

### 4.3 ✅ 设定内容本身
抽查 [01-总纲](../worldbuilding/01-总纲.md)（公设/规则/红线）、[08 头部](../worldbuilding/08-天网：机械降神与拜占庭崩溃.md)、[18 全文](../worldbuilding/18-天网起源：从天上的网到神谕之神.md)，**未发现自相矛盾**——master-slave 约定真的压住了冲突。
> ⚠ **诚实边界**：这是**抽查**，非全库逐 claim 自洽性扫描。**"全库 claim 级自洽性 sweep"本身是推荐给 ultracode 的独立任务**（见 §6-T7），不要假定本报告已证明全库无矛盾。

## 5. 治理重构 backlog（按 ROI 排序）

> 每项给：**问题 → 方案 → 风险 → 可逆性 → 验证**。这是给 GPT Pro 的执行清单，也是给 ultracode 的审计项。全部为**行为保持型重构**：设定内容不变。

### T1 🔴 消灭双登记（最高 ROI，直击 §3.1）
- **问题**：00"拟揭露候选"区是"当前主假说"区的手工镜像，必然漂移。
- **方案**（二选一）：
  - **(a) 推荐**：删掉投影区，需要时由脚本从"当前主假说"区 + 一个"揭露面措辞"列生成。少一份手维护文件。
  - **(b) 保守**：保留两区，但在 validator 加成对标签校验（成对标签必须同时存在/同时缺失）。
- **风险**：低。两区内容已声明等价，删除不丢信息。
- **可逆**：完全可逆（git）。
- **验证**：删除后全文检索原"拟揭露候选"行的关键 claim，确认在"当前主假说"区都有对应行。

### T2 🔴 建"权威表"（Authority Table），把引用环拍成 DAG（直击 §2/§3.3）
- **问题**：一条事实散落多处定义，"以 X 为准"散在各文档头部，写手要重读全图。
- **方案**：新建一张表 `事实 → 唯一 canonical 定义点（文档·稳定节名）`。所有其他出现处只能是指针。把 08 **显式声明为 hub**。
- **风险**：中（需逐 claim 判定归属）。
- **可逆**：可逆，但工作量随激进度上升。
- **验证**：每条事实在表中有且仅有一个权威行；validator 扩展为"非权威处不得出现完整定义"（可选，较难自动化，至少人工 checklist）。

### T3 🟡 散文护栏去重（直击 §3.2）
- **问题**：同一边界约束在 00 边界行 / 文档头部纪律 / 段内 inline 三处复述。
- **方案**：每条约束选**唯一权威住所**（建议统一收进 00 矩阵边界行），文档内只留一句指针（如"边界见 00·边界·终端"）。删掉第二、三遍 inline 复述。
- **风险**：中（要确认每处复述确实等价后才能删，不能删掉了独有信息）。
- **可逆**：可逆。
- **验证**：删除前后，把每条被删复述与保留的权威住所做语义对照，确认无独有信息丢失。18/19 体量应明显下降。

### T4 🟡 把手维护纪律变成校验门（巩固 T1/T3，防回潮）
- **问题**：分层纪律（01-04 只能写共享框架）、双登记同步，目前全靠人脑。
- **方案**：能 lint 的接进 [validate-doc-governance.js](../../tools/validate-doc-governance.js)：
  - 双登记成对标签校验（若 T1 选 (b)）；
  - 01-04 文档内不得出现"假说级/支线真相"标签的简单正则门；
  - 可选：权威表的"唯一定义点"软校验。
- **风险**：低（纯增量校验，不改内容）。
- **可逆**：可逆。
- **验证**：`node tools/validate-doc-governance.js` 绿；故意制造一处违例确认能被抓。

### T5 ⚪ 00 矩阵自身瘦身（可选，低优先）
- **问题**：00 已 207 行，治理元数据自身需被治理。
- **方案**：T1 完成后，分区从 4 个降到 3 个；强度图例与防漂移说明可折叠为脚注。
- **风险/可逆**：低 / 可逆。

### T6 ⚪ 激进档：重排文档拓扑（仅在用户选激进时做）
- **方案**：合并/拆分文档，把 08 设为强制 hub，外围文档间禁止互引（只准引 hub）。
- **风险**：**高**（改动面大、需逐 claim 搬运）。**默认不做**，除非用户明确要"以不留债务"的最高标准重排。

### T7 🟢 全库 claim 级自洽性 sweep（推荐交给 ultracode）

## 6. "不留债务"验收契约（整理过程本身的护栏）

> 这是给 GPT Pro 的**元护栏**——整理动作必须满足以下不变量，否则就是在用新债换旧债。ultracode 复审时逐条核对。

1. **内容冻结**：设定事实（公设/规则/红线/各假说 claim）**一条不增、不删、不改义**。只允许搬运位置、删除重复表达、改写指针。任何看似"顺手修正设定"的改动**必须单独提出、单独评审**，不混入治理整理。
2. **锚点稳定**：现有 `稳定节名`（反引号 `NN·...节`）**不得重命名或删除**——它们是 101 处跨节引用的目标。新增锚点可以，改名/删除会断引用。
3. **validator 全程绿**：每完成一项 T，运行 `node tools/validate-doc-governance.js` 必须通过（尤其"stable section reference missing definition"和"stale anchor leaked"两类）。
4. **可复现指标对照**：整理后重跑 §1 命令，**总行数可降（去重应使其下降），但跨节引用数不应出现悬空**，文档数不应意外变化（除非 T1(a)/T6 显式合并文件，且需在版本历史登记）。
5. **编码纪律**：`.md` 用 UTF-8；直写中文，不要 `\uXXXX` 转义（与仓库 [AGENTS.md](../../AGENTS.md) 硬约束一致）。
6. **治理同步**：若发生文档增删/合并，按 [AGENTS.md](../../AGENTS.md#L23) 文档同步规则更新 README 与 [worldbuilding-version-history.md](worldbuilding-version-history.md)，并重跑 validator。
7. **单一权威**：整理后每条事实只有一个 canonical 定义点（T2 的目标）；不得制造新的"两处都像权威"的歧义——那是新债。
