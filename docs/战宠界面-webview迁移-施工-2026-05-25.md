# 战宠界面 WebView 迁移 — 施工记录 2026-05-25

## 目标

将 `flashswf/UI/战宠相关界面.swf`（宠物信息界面 + 领养宠物界面）从 Flash 迁移为 HTML，通过 WebView2 面板系统被 C# 壳调用。复用现有的 Panel 架构（panels.js + PanelHostController + Task 桥接）。

## 架构

```
JS (pet-panel.js / pet-data.js)  ←→  C# (PetTask.cs)  ←→  AS2 (PetPanelService.as)
         ↕                                    ↕
    panels.js / Bridge.send()           XmlSocketServer / MessageRouter
```

- **静态数据**：`pet-data.js` 包含 110+ 宠物定义、9 分类、25 种进阶方案（从 `data/merc/pets.xml` 和 `单位函数_aka_战宠进阶.as` 提取）
- **运行时数据**：通过 `petSnapshot` 命令从 AS2 `_root.宠物信息` 查询
- **写操作**：领养、出战/休息、进阶升级均经 C# PetTask 桥转发到 AS2 执行

## 踩坑记录

### 1. UTF-8 BOM 缺失
- **现象**：PetPanelService 所有命令 timeout
- **原因**：新建的 `.as` 文件缺少 `EF BB BF` BOM，Flash 编译器静默跳过
- **修复**：`PetPanelService.as` 和 `战宠系统_WebView.as` 添加 BOM

### 2. `cat.ids` 属性名错误 → `cat.List`
- **现象**：打开领养商店后 UI 卡住（10s timeout 后恢复）
- **原因**：`_root.宠物商城列表[c].List` 是 XML 解析后的二维数组（按 `<List>` 行分组），代码中误写为 `.ids`
- **修复**：三层嵌套遍历（行→列→petId），过滤 null

### 3. 领养槽位逻辑错误
- **现象**：扩展格子后领养永远失败
- **原因**：`handleExpandSlot` 调用 `_root.开宠物格子()` 同步 push 空数组，但 `handleAdopt` 又 `push` 新条目，导致 `宠物信息.length >= 宠物领养限制` 恒为 true
- **修复**：改为查找空槽位（`slotEntry.length == 0`）并赋值，而非追加

### 4. .NET Framework 4.6.2 目标包缺失
- **现象**：MSBuild 报 `MSB3644: .NETFramework v4.6.2 引用程序集未找到`
- **环境**：该机器安装了 4.6.1 和 4.7.2，跳过了 4.6.2
- **修复**：`mklink /J v4.6.2 → v4.6.1`（4.6.2 与 4.6.1 API 兼容）

### 5. PetTask.cs 未加入 .csproj
- **现象**：MSBuild 报 `CS0246: 未找到类型或命名空间名称 PetTask`
- **原因**：经典 MSBuild 项目格式需显式 `<Compile Include>`
- **修复**：在 `.csproj` 中添加编译条目

### 6. C# HandlePanelMessage 未覆盖 pet 专用命令
- **现象**：点击"领养宠物"后界面无反应，`_storeLoading` 永久卡死
- **原因**：switch 只覆盖了 `snapshot` 等面板通用命令，`adopt_list`、`adopt`、`deploy` 等 8 个 pet 专用 cmd 不在任何 case 中，消息在到达 PetTask 前被静默丢弃。PetTask 的 10s timeout 从未触发
- **修复**：将 8 个缺失命令加入 case 列表

### 7. 进阶方案 gold 值大面积转录错误
- **现象**：pet-data.js 中 25 个进阶方案 21 个 gold 值与 AS2 源文件不一致
- **原因**：从 AS2 `消耗金币` 转录为 JS `gold` 时未逐项核对
- **修复**：对比 `单位函数_aka_战宠进阶.as` 逐一纠正

### 8. 体力上限 20 → 200
- **现象**：界面显示 `体力: 20/20`，实际应为 `20/200`
- **原因**：`PetPanelService.as` 中 `maxStamina = 20` 转录错误，JS 端又硬编码了 `/20` 而未使用传来的 `maxStamina` 字段
- **修复**：AS2 改为 200，JS 改为动态读取 `pet.maxStamina`

## 2026-05-26 — Fix & Enhance

### Fix: K点数量始终为0

**根因**：`PetPanelService.as` 中多处使用了 `_root.K点`，但该变量在整个代码库中从未被赋值。游戏实际的 K 点存储变量是 `_root.虚拟币`（在 `SaveManager.as` 存/读、`PickUpManager.as` 拾取累加、`商城系统_WebView.as` 商城界面均已使用该变量）。

`Number(undefined) || 0` → 永远为 `0`，导致：
- snapshot 返回 `kpoint: 0`
- 领养检查 `_root.K点 < kprice` 恒为 `true`（0 < 任何正数）
- K点购买的宠物必然显示"K点不足"

**修复**：`PetPanelService.as` 中 5 处 `_root.K点` → `_root.虚拟币`（快照读取/领养检查/领养扣除/响应返回 × 3）。

### Fix: 战宠经验值始终显示 0/--

**根因**：经验值在游戏中的实际存储路径为 `宠物信息[i][5].宠物升级经验` 和 `宠物信息[i][5].宠物升级所需经验`（参见 `引擎_lsy_等级与经验值.as:122-133`）。`PetPanelService.as` 错误地：
1. 从 `info[3]` 读取当前经验——该索引始终为 0，全代码库无任何赋值
2. 用 `calcXpForLevel()` 简化公式估算所需经验，而非读取游戏实际计算值

**修复**（`PetPanelService.as` `handleSnapshot`）：
- `xp: Number(info[3])` → 改为从 `info[5].宠物升级经验` 读取，未进过战斗的宠物默认 0
- `xpNeeded = calcXpForLevel(...)` → 优先读取 `info[5].宠物升级所需经验`，不存在时 fallback 到公式计算
- `宠物升级经验`、`宠物升级所需经验` 这类字符串 key 不会与进阶 scheme 名冲突（for-in 中 `typeof val == "object"` 过滤掉简单数字/字符串值）

### Fix: 领养宠物初始体力 20 → 200

`handleAdopt` 中创建新宠物条目时 `stamina = 20`，与 `maxStamina = 200` 不一致。领养后体力显示 `20/200`，需要额外恢复体力才能使用。

**修复**：`PetPanelService.as` 中 4 处初始体力 `20` → `200`（handleAdopt + handlePreviewAdvance ×2 + handleTooltip 的 fallback context），与 maxStamina/handleRestoreStamina 统一为 200。

## 2026-05-26 下午 — 战宠灵石升级 & 删除战宠

### Feature: 战宠灵石升级（强化按钮）

消耗战宠灵石提升战宠等级 1 级。灵石消耗公式与 Flash 完全一致：

```
消耗灵石 = 当前等级 * 2 + floor(当前所需经验 / 10000)，最低 1
```

**改动范围**：

| 文件 | 变更 |
|------|------|
| `PetPanelService.as` | 新增 `handleLevelUp`：校验等级上限→lazy init 经验值→扣灵石(`singleSubmit`)→等级+1→重算所需经验→刷新出战单位→sendResponse |
| `PetTask.cs` | +2 行，`level_up` → `petLevelUp` |
| `WebOverlayForm.cs` | +1 行，`level_up` case |
| `pet-panel.js` | 进阶页 header 新增"强化"按钮；`renderAdvancePage` 中动态计算灵石消耗 + 满级 disabled；`onLevelUp()` 处理异步回包 |
| `panels.css` | 新增 `.pet-levelup-btn` 紫色系按钮样式 |

**踩坑**：`宠物升级加载()` 的参数是 **宠物mc库索引** 而非 slotIndex，需在 `出战宠物id库` 中搜索对应的 mc 位置。

### Feature: 删除战宠

永久删除战宠并返还部分战宠灵石。返还公式与 Flash 完全一致：
