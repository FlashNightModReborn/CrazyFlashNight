## 0. 背景与四约束

当前 `BulletQueueProcessor` 把**命中判定**（几何扫描 / 窄相）与**伤害结算**（闪避→伤害→事件→飘字→特效）**完全内联交织**在两条独立路径里，没有中间「命中记录」数据结构。本设计要把二者拆成两条管线：**命中管线产出干净命中记录 → 伤害管线消费**。驱动这次拆分的四条约束：

1. **射线逻辑混杂**：`processRayBullets()`（[L564-1728](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Queue/BulletQueueProcessor.as#L564-L1728)）的 5 个模式（single/pierce/chain/fork/combo）已经把伤害结算抽成 `settleRayHit()`，而主循环把同一套逻辑内联——两份实现已分叉且有刻意行为差异，维护脆弱。
2. **绕过碰撞直接出伤**（如 `isLockon`）：未来需要"不经几何检测、直接对目标出伤"的能力。命中/伤害分离正是这一能力的自然实现。
3. **碰撞检测迁 C#**：碰撞判定的输入/输出数据干净（有序数值数组），适合迁移；伤害结算依赖活体对象图 + 事件总线 + 池化，"脏"，应留 AS2。拆分后 C# 只需回吐命中记录，AS2 伤害管线原样消费。
4. **双管线已部分存在**：`settleRayHit` 是伤害管线雏形，射线侧"先收集命中数组、再逐个 settle"是命中/伤害两段式的事实实现——拆主循环有现成蓝本，不是从零设计边界。

### 0.1 反幻觉地基（已对抗验证的 load-bearing 事实）

> 写代码 / 改设计前必读。每条都标了证据，可就地验算。

- **`settleRayHit` 已是伤害管线雏形**：签名 `settleRayHit(ctx, hitTarget, hitX, hitY, dmgMult, tEntry)`，消费命中五元组，内部完成 dodge→damage→hitCount→hit→kill/death→triggerDisplay（[L483-536](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Queue/BulletQueueProcessor.as#L483-L536)）。
- **命中/伤害边界已结晶**：射线 pierce/chain/fork 先把命中拷成 plain 对象数组 `{target,hitX,hitY,tEntry,zOffset}`（[L1258](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Queue/BulletQueueProcessor.as#L1258) 等），再循环喂 `settleRayHit`。这套"元组数组 + 消费函数"就是拆分目标形态。
- **伤害黑箱**：`DamageCalculator.calculateDamage(bullet, shooter, hitTarget, overlapRatio, dodgeState)` 仅以五元组为输入、`DamageResult` 为返回，**零读取 BulletQueueProcessor 循环局部状态**（sweepIndex/killFlags/unitIndex 在 Damage 目录零出现）。可零修改抽出。
- **hit 订阅者只读 `overlapCenter.{x,y}`** `[已验证]`：全代码库 hit 事件订阅者中，唯一读 `collisionResult` 的是 `HitUpdater.doHitUpdate`，且只读 `overlapCenter.x/y`（[HitUpdater.as:77-79](../scripts/类定义/org/flashNight/arki/unit/UnitComponent/Updater/HitUpdater.as#L77-L79)）；`damageResult` 仅 `dodgeStatus` 被订阅者读。**合成 CollisionResult 不存在订阅者侧阻断点**。
- **性能成本带**（[as2-performance.md](../agentsDoc/as2-performance.md)）：函数调用 485ns（高税）、方法调用 1340ns / `new` 847ns（极重）、成员读 150ns、数组索引 35ns。**抽 settleHit 的边际成本 = 1 个函数帧 ≈ 485ns/命中**。
- **霰弹值 NaN 陷阱**：`__commitDeferScatter` 用 `sc < -65535`（严格 `<`）规避 AS2 `NaN <= x` 恒 true 陷阱。重构**禁止**改成 `<=`（[L2600-2602](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Queue/BulletQueueProcessor.as#L2600-L2602)）。
- **AS2 硬约束**：`.as` 必须 UTF-8 BOM、禁从零 Write 新建（丢 BOM 被 `#include` 时编译器静默跳过、`compiler_errors` 仍报 0 错）；新建须 `cp` 现有 `.as` 改名。本设计若新增类（如 `BulletHitRecord`）走复制流程。

## 1. 现状真实耦合

### 1.1 两条路径，两份伤害结算

| | 主循环 `processQueue()` | 射线 `processRayBullets()` |
|---|---|---|
| 入口 | [L1768-2589](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Queue/BulletQueueProcessor.as#L1768) | [L564-1728](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Queue/BulletQueueProcessor.as#L564) |
| 队列 | `BulletQueue`（逐帧移动弹） | 独立 `_rayBullets` 数组（单帧检测即销毁） |
| 命中/伤害 | **完全内联交织**（[L2247-2416](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Queue/BulletQueueProcessor.as#L2247) 单循环内：判定→闪避→damage→publish→死亡→飘字→killFlags） | **天然两段式**（扫描收集命中数组 → 逐个 `settleRayHit`） |
| 命中记录 | 无中间结构（零分配、复用静态 `_aabbResult`） | plain 对象数组（脱离单例，可延迟消费） |
| 绕碰撞注入 | **无此能力** | `isLockon` 已实现（5 处注入/覆盖） |

> **透明弹归属澄清**：透明弹（纯 `Object` 轻量弹）经 `preCheckTransparent` → `add` 进入常规 `BulletQueue`（[L356](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Queue/BulletQueueProcessor.as#L356)），与 MovieClip 弹**同走主循环 settleHit**；只有射线弹经 `preCheckRay` 进 `_rayBullets` 走两段式。故"主循环侧 = MovieClip 弹 + 透明弹"，两路径表无第三条遗漏。

### 1.2 伤害结算骨架（两边共享）

闪避 → `calculateDamage` → `hitCount += actualScatterUsed` → `publish("hit")` → 死亡事件 → `triggerDisplay`。这六步两边逐行同构，是统一 `settleHit()` 的核心。

### 1.4 隐藏的"伤害乘数恒为 1"陷阱 `[已验证：V1 designImplication]`

主循环 AABB 命中走宽相内联，命中后取静态 `_aabbResult`（= `AABBCollider.result`）**只重写 `overlapCenter.x/y`，从不触碰 `overlapRatio`**——它沿用构造时写死的常量 `1`（[AABBCollider.as:67](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Collider/AABBCollider.as#L67)）。即**今天 AABB 子弹的伤害乘数恒为 1**；只有多边形窄相（[L2331](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Queue/BulletQueueProcessor.as#L2331)）才产真实重叠比，射线用 `dmgMult`。

> ⚠ **隐性数值回归红线**：若统一命中记录对 AABB 命中也填"真实 overlapRatio"，AABB 子弹伤害会整体改变——**编译不报错、订阅者不报错、伤害值静默漂移**。byte-equivalent 重构必须复刻"AABB→1 / polygon→真实比率 / ray→衰减"三态。

## 2. 决策结论

1. **拆，且现在拆**：需求成立。代码库已半完成（`settleRayHit` 雏形 + 射线两段式 + `isLockon` 旁路范式）。这是维护性净收益 + C# 迁移的前置去风险步骤。
2. **流式拆分，不物化**：命中管线在"确认命中那一行"直接调 `settleHit(...)`，**零中间分配**——和 `settleRayHit` 今天被调用的方式一致。**禁止**为主循环常见的"单发命中即消失"弹强行物化一个命中记录数组（那是唯一的真性能回退，见 §7）。
3. **统一 `settleHit()`，合并两份**：把主循环内联块**仅逐命中结算段**（[L2354-2397](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Queue/BulletQueueProcessor.as#L2354)，止于 `triggerDisplay`）与 `settleRayHit` 合并成单个函数，六处分歧（§1.3）全部参数化。**抽取边界止于 L2397** —— 其后的近战硬直 + `killFlags`（[L2399-2405](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Queue/BulletQueueProcessor.as#L2399)）是循环控制信号、由 `killFlags` 局部变量在循环尾消费，**必须留在主循环调用点、不进 settleHit**（见 §3.3）。终态 = **2 个命中生产者 + 1 个共享伤害消费者**。
4. **分期施工**：先做 AS2 内部流式拆分（无悔、可独立验收），再做 `isLockon` 旁路坍缩，**最后**才评估 C# 碰撞迁移（重、需架构决策，见 §6）。

## 3. 目标架构

### 3.1 数据流

```
┌─────────────── 命中管线（HitPipeline，热路径，O(B+T)，可迁 C#）───────────────┐
│  消弹 token 否决  →  扫描线宽相  →  窄相(AABB内联 / polygon懒加载 / ray Slab) │
│                              │ 确认命中                                       │
│                              ▼ 流式调用（零分配）                             │
│         settleHit(ctx, target, collisionResult, dmgMult, tEntry, fireTrigger)│
└──────────────────────────────┬───────────────────────────────────────────────┘
                               ▼
┌─────────────── 伤害管线（DamagePipeline，冷路径，O(hits)，留 AS2）──────────┐
│  闪避 → calculateDamage(黑箱) → hitCount累加 → publish hit                   │
│       → kill/death/enemyKilled(_killed守卫) → triggerDisplay                  │
└──────────────────────────────────────────────────────────────────────────────┘
                  返回 DamageResult；hitCount 经 bullet 字段副作用累加、
                  killFlags 由调用方在 settleHit 返回后于循环局部置位
                               ▼
            循环控制（命中管线尾）：pierceLimit break · 命中FX聚合
              · killFlags收尾 · updateMovement · 威胁预警 · 单出口销毁
```

### 3.2 统一 `settleHit()` 契约

> **§4.0 (a) 决策态（下方伪代码已同步；施工以此为准）**：`settleHit` 收 caller 算好的 `dodgeState`（**替换**原 `fireOnHitTrigger`），**本体不内算闪避、不调 `击中时触发函数`、不做 `附加层伤害计算=0`**——这三件留各 caller 原位（main：[L2358-2370](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Queue/BulletQueueProcessor.as#L2358)，闪避→触发顺序不变；ray：`hitIndex==0` 的 per-hit 循环前）。settleHit 职责收窄为：同步命中引用(幂等) → `calculateDamage` → `hitCount+=` → `publish("hit")` → kill/death(带 `_killed` 守卫) → `triggerDisplay`。零分配 / ctx 复用 / collisionResult 入参 / NULL 哨兵 / `hitCount` 走 bullet 字段等契约不变。

```actionscript
// 私有热点路径方法。沿用 settleRayHit 的零分配范式：ctx 走 static 复用对象，
// collisionResult/damageResult 作为已物化引用入参，函数体内禁止 new / 对象字面量。
private static function settleHit(
        ctx:Object,                  // per-bullet 不变量（每发子弹入口 field 赋值一次，非逐命中 new）
        hitTarget:MovieClip,
        collisionResult:CollisionResult, // 命中几何；hit 事件透传；最小字段 overlapCenter.{x,y}
        dmgMult:Number,              // 独立伤害乘数：AABB=1 / polygon=overlapRatio / ray=衰减乘积
        dodgeState:String            // §4.0(a)：caller 已算好的闪避态（替换原 fireOnHitTrigger）；settleHit 不内算闪避
        ):DamageResult               // tEntry 不入参：caller 形成 collisionResult 时已写入(§3.2补注①)，settleHit 不读
{
    // 0. 从 ctx 解出 per-bullet 不变量（与 settleRayHit L488 一致，bullet 不单独入参以省一个参数槽）
    var bullet:Object = ctx.bullet;
    // 1. 同步命中引用（幂等；caller 通常已设 命中对象 供其 击中时触发函数 读取，此处再设无害）
    bullet.hitTarget = hitTarget; bullet.命中对象 = hitTarget;
    // 2. 【§4.0 (a)：不在 settleHit】首命中副作用（击中时触发函数 + 附加层伤害计算=0）由各 caller 在原位执行：
    //    main = L2358 重置 / L2370 触发（闪避后、触发前）；ray = hitIndex==0 的 per-hit 循环前。
    //    两侧 byte-order 因此不变：main 闪避→触发→伤害；ray 触发→闪避→伤害。
    //    （附加层伤害计算=0 写不可省：4 个敌人帧脚本"受击反制"读它做击倒阈值，漏写→undefined→NaN→击倒失效——责任在 caller）
    // 3. 闪避：dodgeState 由 caller 算好经入参传入，settleHit 不再调用 calculateDodgeState（无重排）。
    // 4. 伤害（黑箱，dmgMult 作第4参，不从 collisionResult 硬读）
    var damageResult:DamageResult = ctx.Damage.calculateDamage(bullet, ctx.shooter, hitTarget, dmgMult, dodgeState);
    bullet.hitCount += damageResult.actualScatterUsed;
    // 5. hit 事件（collisionResult 透传，订阅者只读 overlapCenter）
    var d:Object = hitTarget.dispatcher;
    d.publish("hit", hitTarget, ctx.shooter, bullet, collisionResult, damageResult);
    // 6. 死亡事件（统一带 _killed 守卫 —— D1 裁决：普适，对射线无害对主循环必需）
    if (hitTarget.hp <= 0 && !hitTarget._killed) {
        var isNormalKill:Boolean = (ctx.flags & ctx.meleeMask) == 0; // 从 ctx.flags 取（已 |0 规整，与 settleRayHit L522 一致）；勿绕过 ctx 直读 bullet.flags
        d.publish(isNormalKill ? "kill" : "death", hitTarget);
        ctx.shooter.dispatcher.publish("enemyKilled", hitTarget, bullet);
    }
    // 7. 飘字
    damageResult.triggerDisplay(hitTarget._x, hitTarget._y, hitTarget._name);
    return damageResult;   // 仅为 injectHit/未来扩展预留；当前两个生产者的循环控制都不读返回值
    // 注意：循环控制不依赖返回值 —— hitCount 走 bullet 字段共享（步骤4，勿局部化）、
    //       killFlags/近战硬直由调用方在返回后于循环局部置位（D3/§3.3）
    // 注意：命中特效(击中后子弹的效果)不在此 —— D3 裁决：归命中管线，两套语义各自处理（见 §8 阶段2 的 10 个射线调用点补偿清单）
    // 注意：DamageResult 可能为 NULL 哨兵（无敌/man.无敌标签/NPC，DamageCalculator L97；或 hp<=0 L105——
    //       但 hp<=0 已被 §5.6 hp>0 legality 过滤，仅 击中触发函数 中途致死的罕见边界才到此）。
    //       步骤 5/7（publish hit / triggerDisplay）仍须无条件执行（NULL 空列表使其幂等），禁止用 damageResult!=NULL 门控跳过。
    //       ⚠ 步骤 4 的 hitCount += actualScatterUsed 在 NULL 路径下 = +1（NULL.actualScatterUsed=1，DamageResult.as:165 reset 设、NULL 不可变）
    //          → 无敌/NPC 命中**会消耗 1 个 pierce 配额**(L2411)并满足命中FX门控(L2419)。拆分**沿用现状不静默改**；
    //            是否应让 无敌/NPC 吃配额属独立 gameplay 决策，见 §10。
}
```

## 4. 行为差异裁决表

### 4.0 顺序决策（2026-06-22 修订 · 覆盖下表 D-order 行、§3.2 步骤 2/3、§8 阶段1③、§10 对应项）

**结论：采用保守序 (a) —— `settleHit` 不内算闪避、不调 `击中时触发函数`；两侧各自保留原 byte-order。** 据此：

- `settleHit` 签名去掉 `fireOnHitTrigger:Boolean`、改收 caller 算好的 `dodgeState:String`；`击中时触发函数` + `附加层重置` 留在各 caller 原位（**main**：[L2358/L2370](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Queue/BulletQueueProcessor.as#L2358) 闪避后、settleHit 前；**ray**：`hitIndex==0` 的 per-hit 循环前）。两侧顺序均不变：main = `闪避→触发→伤害`，ray = `触发→闪避→伤害`。下表 D-order 由"真实重排"降级为**零变更**。`settleHit` 职责收窄为"消费 `dodgeState` → 伤害→hit→死亡→飘字"。
- **性能依据（这是 (a) vs (b) 的决定性比较）**：两方案逐命中操作数**完全相同**——同样 1 个 `settleHit` 函数帧（485ns）、同样 2 次 dodge 方法帧、同样 1 次触发帧，无额外 `new` / 分支；闪避与触发只是**词法位置**不同（settleHit 内 vs caller），调用次数与成本不变（§7 只把成本归于"抽函数本身"，对 (a)/(b) 等价）。**性能平局** → 以风险破平局：(a) 不把任何触发副作用暴露给本命中的闪避计算，零回归面，故选 (a)。
- **被推翻的论据**：原 D-order"低风险"依据"`击中时触发函数` 仅发射新子弹"**经核实为假**——[毒液蜘蛛侠.as:37](../scripts/逻辑/装备函数/毒液蜘蛛侠.as#L37) / [雾人:115](../scripts/逻辑/单位函数/单位函数_雾人_aka_fs_主动战技.as#L115) 给目标加 buff + 停弹 + `gotoAndPlay("爆炸")`，[键盘镰刀:202](../scripts/逻辑/装备函数/键盘镰刀.as#L202) 让射手硬直。该假命题源于源码注释 [L2617](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Queue/BulletQueueProcessor.as#L2617) 的 2026-03 审计（原范围仅 C2"不写 `霰弹值`"，被过度泛化）+ [ShadowBookkeeping.md:169](../scripts/类定义/org/flashNight/arki/component/Damage/ShadowBookkeeping.md#L169)。采用 (a) 后此假命题不再 load-bearing；上游两处注释的收窄列为**阶段0 行动项**（§8 阶段0）。

## 5. 不变量与回退红线

> 拆分若违反任一条 = 静默回归（编译 0 错、可能真机才暴露）。施工时逐条对照。

### 5.1 霰弹值影子记账窗口 C1-C4 `[已验证：V2 = confirmed]`

- **窗口仅存在于主循环、由 `FLAG_CHAIN` 门控**：`__initDeferScatter`（[L2179-2181](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Queue/BulletQueueProcessor.as#L2179)）/ `__commitDeferScatter`（正常出口 [L2423](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Queue/BulletQueueProcessor.as#L2423) + 空窗早退出口 [L2219](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Queue/BulletQueueProcessor.as#L2219)）。
- **红线①**：`init/commit` 必须**留在主循环调用点、不下沉进 settleHit 函数体**。否则会对每个射线 settleHit 各开一对窗口，破坏"一发子弹一窗口、跨多命中累计扣减"（INV-2）。
- **红线②**：`settleHit` **不可被无条件复用到无窗口的射线路径**。射线侧 `霰弹值` 恒正，`MultiShotDamageHandle` 走直接模式（[MultiShotDamageHandle.as:122](../scripts/类定义/org/flashNight/arki/component/Damage/MultiShotDamageHandle.as#L122)）。若日后真要统一，把"是否处于影子窗口"参数化（`ctx.shadowWindowed`）。
- **红线③**：命中管线**绝不触碰 `bullet.霰弹值`**（C2 独占写）。窗口内裸读得到负编码（C3）。任何新增 processQueue 出口（continue/break/return 到迭代外）必须配 `commit`（C4），否则下帧 [联弹管理](../scripts/逻辑/战斗系统/战斗系统_fs_联弹管理.as) / [ShellSystem.as:167](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Shell/ShellSystem.as#L167) 读到负编码崩溃。

### 5.2 命中即结算的同步性 / 单例复用安全

复用单例 `_aabbResult`（[AABBCollider.as:67](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Collider/AABBCollider.as#L67)）/ `_rayHitResult` / `DamageResult.getIMPACT()` 依赖**同步事件分发**（publish 同步跑完订阅者再改写下一命中）。

- **红线**：流式 settleHit 满足此约束（逐命中即时消费）。但若未来引入"先收集全部命中再批量结算"或 C# 异步回灌，复用单例会被后续命中覆盖 overlapCenter/overlapRatio → 订阅者读错坐标、`DamageResult` 被覆盖。届时命中记录必须**值拷贝脱离单例**（射线 pierce 已是此范式：拷成 plain 对象数组）。

### 5.3 终止意图与销毁分离

命中结算只置 `killFlags` 位声明终止意图（[L2400-2415](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Queue/BulletQueueProcessor.as#L2400)），真正 `removeMovieClip`/`gotoAndPlay("消失")` 在循环尾单出口按四优先级执行（[L2536-2575](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Queue/BulletQueueProcessor.as#L2536)）。碰撞器回收延迟到 `BulletLifecycle.bindSafeRemove` 重写的 `removeMovieClip`（[BulletLifecycle.as:97-151](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Lifecycle/BulletLifecycle.as#L97)）。

- **红线**：伤害管线产出必须是**位标志/意图**，不是终态副作用。命中记录消费函数若直接销毁子弹 = 破坏穿透多命中 + 消失动画期提前回收碰撞器 → AABB 污染回归。

### 5.4 消弹 token 前置否决

消弹是"命中判定之前"的否决闸：`mode3(BOUNCE)`→整弹跳过命中；`mode1/2(VANISH/REMOVE)`→`skipUnits=true` 短路整条命中循环 + 威胁尾循环（[L2160-2176](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Queue/BulletQueueProcessor.as#L2160)）。

- **红线**：命中管线必须把消弹 token 读取/否决放在**命中判定最前、伤害管线之前**。否则被消弹的弹会先产出命中记录再被撤销，bounce 弹更不该进命中循环。

### 5.5 暂停冻结语义

`processQueue` 首句即暂停闸门（[L1770-1783](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Queue/BulletQueueProcessor.as#L1770)）：`reset 消弹 + clear 全队列 + _rayBullets.length=0 + return`，且 `processRayBullets()` 在闸门**之后**调用。

- **红线**：暂停 = "清空两条队列 + 清空射线缓冲 + 复位消弹"，**不是跳过一帧**。任何新管线入口（尤其 C# 回灌驱动）必须落在闸门内并复刻清空，否则暂停 = 积压而非冻结。

### 5.6 命中目标合法性单点校验（含自碰撞剔除）

> 原"自碰撞剔除"与"合法性校验"两条合并为本条单一 legality 不变量，消除"伤害管线复核 vs 不复查"的歧义。

命中判定阶段的合法性过滤共三项：`hp>0`、`防止无限飞!=true`、`hitTarget !== bullet`（主循环 [L2280](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Queue/BulletQueueProcessor.as#L2280)，注释 [L2277](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Queue/BulletQueueProcessor.as#L2277) "防御性自碰撞剔除"）。

- **`hitTarget !== bullet` 语义**：UnitBullet（可拦截单位子弹）同时存在于 BulletQueue（作子弹）和 TargetCache（作单位），阵营路由已保证同阵营不交叉，此为零开销指针比较的**安全兜底**。**射线路径安全省略它**（[L1266](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Queue/BulletQueueProcessor.as#L1266) 等无此项）——射线弹（FLAG_RAY）不是单位、不进目标缓存，永不等于自身目标，故省略无害（**与 `_killed`（D1）不同，这不是潜在 bug**）。

- **归属裁决（消歧）**：三项合法性过滤**全部归命中管线（producer）**，伤害管线/settleHit **默认不复查**（避免冗余成员读）。**唯一例外**：对 C# / `injectHit` **外部注入**的命中，settleHit 保留**一个** `hitTarget !== bullet` 零开销指针兜底——因外部生产者可能不知道某 target 同时是正在结算的 bullet 本体。即 `hitTarget !== bullet` 是 **HitPipeline 的生成责任**，C# 注入侧须用子弹 uid/_name 排除自身；settleHit 内的指针比较仅作"注入命中"的最后一道兜底，**不是对每发命中的常规复查**。

### 5.7 `additionalEffectDamage` 跨命中躲闪反馈环（与霰弹值同类的隐式契约）

这是一个 **bullet 作用域、跨命中累加**的状态，与霰弹值窗口同类但更隐蔽：伤害管线写（[CrumbleDamageHandle.as:94](../scripts/类定义/org/flashNight/arki/component/Damage/CrumbleDamageHandle.as#L94) / [NanoToxicDamageHandle.as:108](../scripts/类定义/org/flashNight/arki/component/Damage/NanoToxicDamageHandle.as#L108) 在 `calculateDamage` 内 `bullet.additionalEffectDamage +=`），**下一次命中的闪避管线读**（[DodgeHandler.as:114](../scripts/类定义/org/flashNight/arki/component/StatHandler/DodgeHandler.as#L114) 在 `懒闪避>0` 路径 `rawHit = (子弹威力|0) + (additionalEffectDamage|0)`），**生命周期级重置**（[BulletLifecycle.as:41](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Lifecycle/BulletLifecycle.as#L41) `=0`，非逐命中）。

- **红线①**：命中管线 / settleHit **不得局部化或重置** `additionalEffectDamage`——它必须留在 bullet 实例上跨命中累积。
- **红线②**：同一发子弹的多个命中**必须按命中顺序串行喂伤害管线**（与霰弹值 INV-2 同源）。迁 C# 异步回灌 / 批量 drain 若打乱同弹命中顺序，hit N+1 的懒闪避会读到错误累计值 → 闪避概率漂移。§6.3 硬约束须把它与霰弹值并列。

## 6. C# 注入命中边界（前瞻，分期，含可行性判断）

> 这是约束 3 的落点。**结论先行**：作为"伤害管线 drain 命中记录批"的消费端**现实可行**且与现有 `hn` 伤害数字批同构；作为"实时同帧碰撞替换"**不现实**——当前 socket 是单向帧末广播 + 异步回灌，无同帧请求/响应能力。

### 6.1 现有 AS2↔C# 通路

- **通道**：单客户端 TCP `XMLSocket`（[XmlSocketServer.cs](../launcher/src/Bus/XmlSocketServer.cs)），`\0` 分隔，`NoDelay=true`，快车道前缀（F/R/S/B/N/W/U/D）绕 JSON 零 GC。
- **节奏**：AS2 每帧帧末由 `FrameBroadcaster.send()` 发一条 `F` 消息（载荷 = 摄像机 / `hn` 伤害数字批 / fps / UI / 输入位掩码，全是**表现层 + 输入层**扁平文本）；C# 用 `K` 前缀回发搓招。**异步、无同帧 ACK**，输入存在 ≥1 帧往返延迟。

### 6.2 干净碰撞输入

`TargetCacheManager.acquireEnemyCache/acquireAllCache` → `SortedUnitCache`（并行数组 SoA：`data/leftValues/rightValues/rightMaxValues + nameIndex`，"纯数据模型，无状态依赖"）。宽相判定（双指针扫描线 + `bsearchScanStart` 二分 + AABB 三轴分离 + Z 粗判）是**纯算术、无副作用、确定性、字节级可测**。这是"碰撞输入干净适合迁 C#"的实锤。

### 6.3 切分边界与硬约束

**clean 宽相判定（可迁 C#，输出命中元组批）| dirty 结算（留 AS2，drain 命中元组批）**。

C# 碰撞的硬约束（`[需架构决策]`）：
1. socket 异步无同帧保证 → C# 碰撞引入 **≥1 帧命中延迟**，除非新增同步阻塞通道（当前无，且会卡帧）。
2. C# 做碰撞须每帧把全部 unit+bullet 的 AABB(l/r/t/b)+Z+hp 序列化过桥（当前完全不过桥）= 新增大宗每帧流量，可能抵消迁移收益。
3. 命中元组须用**稳定键**（`_name` 或缓存索引+帧版本戳，参考现有 `_cancelToken` 的 `queueStamp` 编码）跨边界；回传后 AS2 经 `nameIndex` 重解析 `MovieClip`。cache 每帧原地刷新，索引须绑定 `frameId/queueUID`。
4. **per-bullet 顺序消费约束**（两条同弹跨命中累加状态都受此约束）：① 联弹 `霰弹值` 影子窗口（§5.1）须按 `bulletId` 重建 **per-bullet** 窗口（每弹各自 init→消费其全部命中→commit），不能用单一全局窗口，否则 A 弹编码被 B 弹 `calculateDamage` 解码串扰；② `additionalEffectDamage`（§5.7）须保证同弹命中**按顺序串行** drain，否则懒闪避读到错误累计值。异步回灌/按目标聚合的批次都会破坏这两条——回传粒度须是"子弹 × 命中列表"而非"目标 × 命中"。
5. 窄相裁决：旋转联弹的多边形/Slab 窄相留 AS2 还是去 C#，决定命中元组要不要把 `overlapRatio` 作为契约字段回灌（且复刻 AABB=1 差异，§1.4）。

### 6.4 现实可行的近期形态

把 `isLockon` 的 5 处注入坍缩成命中管线的**单一外部命中注入 API** `injectHit(target, hitX, hitY, tEntry, dmgMult) → 产出命中记录 → settleHit`。`isLockon` = 该能力首个**内部**消费者；未来 C# 回灌 = 同一接口的**外部**消费者。二者同构。这一步纯 AS2、无桥改动、可独立验收，是约束 2/3 的共同地基。

## 7. 性能账

| 维度 | 频率 | 单次成本 | 是否瓶颈 |
|---|---|---|---|
| 命中判定（扫描） | O(B+T)/帧，每弹×每候选 | Z 粗判 + AABB 重叠，免费/低税带（0~35ns） | **是**（多弹时帧成本主体） |
| 伤害结算 | O(hits)/帧，仅真命中 | `calculateDamage`(1340ns) + 1~2 `publish`(各 1340ns) + dodge(2×1340ns) + `triggerDisplay`(1340ns) ≈ 6700~9400ns | **否**（命中远比扫描稀疏） |

**抽 settleHit 的边际成本 = 1 个 DF2 函数帧 ≈ 485ns/命中** `[已验证：V4 = confirmed]`，占单次命中成本 5~7%（按成本带地板算）；计入真实全链路（dodge + publish 二级 bus + 订阅者 fan-out）则稳低于 5%。命中事件相对 O(B+T) 扫描稀疏，**净帧成本亚 1%**。

> **补：主循环侧 ctx 填充成本**。今天主循环无 ctx（`Dodge/Damage/FX/shooter` 等是 `processQueue` 内一次性提升的局部变量，内联块直接读）。改调 `settleHit(ctx,…)` 后，主循环须像射线那样填 ctx 字段——但这是 **per-bullet（非 per-hit）**：每发命中弹约 8 次成员写（≈1.2μs），被多命中穿透弹摊薄，不改变"亚 1%"结论。可选优化：主循环 `settleHit` 直接以局部变量逐个传参、不走 ctx 打包，省这笔填充（射线侧因有 5 模式 + 13→6 参降维才用 ctx，主循环未必需要）。

> 🚫 **唯一的真性能回退**：给主循环常见的"单发命中即消失"弹**物化一个命中记录数组**（每弹 `new` 847ns + 第二趟遍历 + GC）。流式 settleHit 不需要它——直接在命中点调函数即可。
>
> **零分配硬条件**（V4 designImplication）：settleHit 沿用 `settleRayHit` 范式——ctx 走 static 复用对象（每发子弹入口 field 赋值，非逐命中 new），collisionResult/damageResult 作已物化引用入参，函数体内**禁止**对象字面量/`new`；函数体须引用参数以触发 DF2(485ns) 而非 DF1(1079ns)；`hitCount/killFlags/shouldGeneratePostHitEffect` 经 bullet 实例/ctx 跨边界共享，勿局部化。若统一两套语义需逐命中携带新"命中记录元组"对象，该元组也必须做成 static 复用（否则 `new`(847ns) 反超 485ns 收益）。

### 9.1 验收采样方式（避免"人工印象判断"）

> `trace()` 在发布设置勾选"省略 trace 语句"后零开销（编译产物完全剔除，[as2-anti-hallucination.md §3](../agentsDoc/as2-anti-hallucination.md)），故可在 settleHit 入口自由插桩；验收完毕后随净化重编移除。

- **插桩点**：在 `settleHit` 入口（及改前的内联块同一位置）`trace` 一行定长字段：`帧号 | bullet._name | hitTarget._name | dmgMult | dodgeState | hp_before | hp_after | 事件序列(hit;kill|death;enemyKilled)`。改前/改后各跑同一脚本，`tools/trace-diff`（asLoader 重构遗留工具）做逐行等价比对——这是事件序列/数值等价的**机器判据**，非肉眼。
- **固定靶场**：用 dev 钩子（仿 `_root.测试角斗场爬升()` 范式）在固定坐标生成 N 个**固定 HP、固定阵营**的靶（含 1 个 UnitBullet 靶验 D1/§5.6、1 个 Gazer 验 §9 项12），关闭随机波动或固定 `PinkNoiseEngine` 种子，使伤害可复现逐发比对。
- **`killStats` 快照**：击杀前后读 `_root.killStats.total` 与 `_root.killStats.byType`（[EnemyKilledEventComponent](../scripts/类定义/org/flashNight/arki/unit/UnitComponent/Initializer/EventComponent/EnemyKilledEventComponent.as)），改前/改后两份快照逐键比对——验 UnitBullet 不重复 kill / `byType` 不按 `_name` 膨胀（项8）。
- **最小复核步骤**：① 固定靶场打一轮；② 收 trace + killStats 快照（baseline，改前）；③ 切换到改后 SWF 重跑同脚本；④ `trace-diff` + killStats 逐键 diff = 空 → Gate 通过；非空 → 定位首条分歧行回归。每个 Gate（1/2/3）复用同一套采样，只换靶场覆盖该阶段的模式（阶段2 加射线 5 模式靶、逐命中点特效目视计数）。
