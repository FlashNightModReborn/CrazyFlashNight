## 数据协议

### snapshot（打开面板时拉取全部任务概要）
- JS → C#: `{type:"panel", panel:"tasks", cmd:"snapshot", callId:"..."}`
- C# → Flash: `{task:"cmd", action:"taskSnapshot", callId:N}`
- Flash → C#: `{task:"task_response", callId:N, success:true, tasks:[{taskId, title, type, npcName}]}`
- C# → JS: `{type:"panel_resp", panel:"tasks", cmd:"snapshot", callId:"...", success:true, tasks:[...]}`

### detail（加载单个任务详情）
- JS → C#: `{type:"panel", panel:"tasks", cmd:"detail", callId:"...", index:N}`
- Flash → JS: `{success:true, task:{taskId, type, title, description, stageReq, itemReqs, npcName, rewards}}`

## 已知问题与修复

### BOM 缺失（已修复）
- **症状**：Flash AS2 编译器跳过无 BOM 的 .as 文件，gameCommands 未注册，数据请求无响应
- **影响文件**：`TaskPanelService.as`、`任务系统_WebView.as`
- **修复**：为两个文件添加 UTF-8 BOM (`\xef\xbb\xbf`)
- **参考**：PetPanelService.as、战宠系统_WebView.as 均以 UTF-8 BOM 开头

### 面板黑屏（已修复）
- **症状**：面板打开后全黑，无任何内容渲染
- **根因**：createDOM 根 div `_el` 未设 `width/height: 100%`，子元素 `.task-panel-container` 的 `height: 100%` 计算为 0
- **修复**：添加 `_el.style.width = '100%'; _el.style.height = '100%';` + `return _el;`

### 数据加载失败 → 左侧正常，右侧无任务数据（5/30 第二次测试）
- **根因**：`handleDetail` 的 AS2 响应中 `task` key 重复：
  ```as2
  sendResponse({ task: "task_response", ..., task: task });
  //             ^^^ 信封字段 = "task_response"    ^^^ 数据对象 — AS2 对象字面量中后者覆盖前者
  ```
  LiteJSON 序列化后 `task` = 数据对象（非 `"task_response"`），C# 的 `msg.Remove("task")` 错误地删除了任务数据，JS 收到的响应无 `task` 字段 → "无任务数据"。
- **修复**（5/30 23:XX）：
  - AS2：`task: task` → `taskData: taskData`（避免与信封 `task: "task_response"` 冲突）
  - JS：`renderTaskDetail(data.task)` → `renderTaskDetail(data.taskData)`
- **对比**：`handleSnapshot` 正常，因数据字段是 `tasks`（复数），不与 `task` 信封冲突。PetPanelService 所有响应数据字段名（`snapshot`、`adoptable` 等）均不与信封 `task` 冲突。

### NPC 头像加载（5/30 23:XX）
- **需求**：从 `flashswf/portraits/profiles/{npcName}.png` 加载 NPC 头像
- **实现**：
  - C#：新增 `TryRegisterGameAssetsVirtualHost`，映射 `cfn-assets.local` → `{projectRoot}/flashswf/`
  - JS：`avatarUrl(npcName)` → `https://cfn-assets.local/portraits/profiles/{npcName}.png`
  - 无对应文件时 `onerror` 回退到 `无头像.png`
  - 左侧任务图标 + 右侧 NPC 详情区域均使用 `<img>` 代替文字占位
