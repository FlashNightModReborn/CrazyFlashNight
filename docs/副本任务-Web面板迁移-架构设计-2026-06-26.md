## 1. 背景与目标

把旧的"副本任务/委托任务"面板从 Flash 迁到 Web，做成现有 **tasks 面板的第 4 个 tab**（我的任务 / 事件日志 / 成就 / **副本任务**），复用任务系统已有的"带立绘迷你对话组件"渲染委托简报对话。

**关键定性**：这是一条**高复用、中等难度**的迁移——
- 数据已在 task 系统里（`data/task/mercenary_tasks.json` 的 44 条 `chain:"委托"`）；
- 委托简报对话与任务回放对话**同结构**，`DialogueView` 可复用；
- 进图函数（扣费 / AddTask / 设限制词条 / 载入关卡）全部现成，**不重写**；
- C# 侧加 cmd case 即可，无需新建 Task。

**唯一真正新建**：一个轻量 AS2 `DungeonPanelService`（承载进图写操作）+ web 第 4 tab + 海报 PNG 资产线。

**和真·对话框迁移的边界**：本面板是**战斗外**的进图前简报，委托对话是线性静态。关卡内、事件驱动的运行态过场（`StageEvent.executeDialogue` → 旧对话框 → `followingEvent`，如 `data/stages/副本任务/密林中央.xml` 的 `<Event><Dialogue>` 边讲边刷怪）**不在本次范围，继续留 Flash**。

## 3. 数据来源（已钉死）

副本 = 任务系统的"委托"链。SOT = `data/task/mercenary_tasks.json`（44 条 `chain:"委托"`），经 `tools/derive-task-catalog.js` 派生进 `launcher/web/modules/tasks/task-catalog.json`。

样本 id 20000「通缉任务之李小龙coser」字段：
```
imageurl: "flashswf/images/wanted_lixiaolong_index.swf"   // WANTED 海报 SWF
get_conversation: "$MERC_LEE_GET"   // 引用 key → data/task/text/mercenary_text.json
get_npc: "酒保"  finish_npc: "酒保"
recommended_level: "16-22"  restricted_level: 8
deposit: 10000  Kdeposit: 0
rewards: ["金币#80000","经验值#10000","复活币#5","情报鸡尾酒#1"]
challenge: { difficulty:"地狱", limitations:["DisableCompanion"] }   // 进入挑战
chain: "委托"
```

**双键陷阱**（MEMORY 已记，活路径勿删）：进图所需关卡字段（关卡路径 / 普通模式 `Limitation` / 起点帧 / 淡出跳转帧）**不在 task 里**，在 `_root.StageInfoDict[stageName]`，`stageName` = task 的中文 `title`。用 `id` 取 task、用 `title` 取 StageInfo，严格区分。

**当前派生器丢弃了副本专属字段**（`imageurl/deposit/Kdeposit/restricted_level/recommended_level/challenge` 在 catalog 里全无）。本设计第 6 节扩展之。

## 5. 迁移闭环

副本视图复用 tasks 面板信封（panel=="tasks" 已注册路由，PanelBridge action/task 守卫已泛化）。**入口由 AS2 NPC 交互发起**（非 web 浏览），故无 browse-all snapshot。

| 触发/cmd | 方向 | 内容 | 写状态 |
|---|---|---|---|
| `openWebDungeon` | **AS2→C#→web 发起** | NPC「获得任务」(`Symbol 1770`) → AS2 `DungeonPanelService` 发 open，携带 `{taskId}` 作为面板 initData（替代旧 `_root.委托任务界面` 打开）；web onOpen 据此拉详情。**这是唯一进入路径** | — |
| `dungeonDetail` | web→AS2，读 | 当前副本详情：普通模式 `Limitation`（StageInfoDict 运行态）+ `challenge.limitations`（catalog）的**原始词条名数组**（web 本地映射成中文）、契约金/K点可负担性、等级门(`restricted_level`)、是否已接 | 读 |
| `dungeonBriefing` | web→AS2，读 | 委托简报对话 lines（复用 `handleReplayDialogue` lines 构建，**去防剧透 gate**——简报是接取前 get_conversation） | 读 |
| `dungeonEnter` | web→AS2，**写** | `{taskId, mode:"normal"\|"challenge"}`。服务端硬门控**复刻 `Symbol 1873.xml:83-148,244-253`**：金钱≥契约金(`deposit`)、K点路径 虚拟币≥`Kdeposit`(扣虚拟币+扣金钱)、等级≥`restricted_level` → `AddTask` + `委托界面进入关卡`(挑战叠加 `challenge.limitations`+`challenge.difficulty`) | **写** |

**回包复用 `task_response` 信封**（不新建 C# Task / TaskRegistry 项）：DungeonPanelService 回包 task 名 = `task_response`，带 `cmd=dungeon*`，JS 在 tasks 面板按 cmd 路由到副本 tab。具体桥接对照 `TaskTask.cs:84-136` cmd switch 与 `TaskPanelService` dispatch，施工时逐项核。**面板形态**：副本 tab 是 tasks 面板第 4 tab（复用视觉外壳）；无 NPC 上下文直接点 tab 显示"前往 NPC 处领取副本"空提示，不可在此浏览/接取其他副本。

**进图上下文重建（dungeonEnter 的核心，已逐字落定）**——`委托界面进入关卡` 读 `this.*`，旧 clip 的上下文由 `显示任务明细`(Symbol 1873.xml:9-82) + `配置关卡属性` 组装，全部源自 `taskData`(=`TaskUtil.tasks[taskId]`)：

## 9. 风险与未知数

1. **WANTED 海报 ffdec 烘焙**：批量 SWF→PNG 的还原度 / 透明 / 体积，需一轮审。
2. **`dungeonEnter` 是写存档 + 场景转换**：后果重（扣钱+进战斗），门控与时序是重点。
3. **双键 `stageName=title`**：id 取 task、title 取 StageInfo，混用即错。
4. **派生器改 catalog**：只新增字段、bump version，须保现有 tasks 面板不破（实测 `--check` + harness）。
5. **`performTaskOpen` 遗留桥**：迁后若不处理，旧 Flash 面板与新面板并存。
6. **委托简报防剧透 gate**：`dungeonBriefing` 必须去 gate（接取前展示），不能直接复用 `replayDialogue`。
