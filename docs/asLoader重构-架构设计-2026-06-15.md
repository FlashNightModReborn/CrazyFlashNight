## 1. 已验证的关键事实（反幻觉地基）

经 14-agent 调查 + 对抗验证 + 独立 SWF 二进制实测，下列事实为本方案地基：

1. **无 per-DoAction 尺寸墙**（原假设被推翻）。asLoader.swf 内已存在单个 **236,169 字节** 的 DoAction（4 个 tag > 64KB），常量池用量远低于 u16 上限 65535。**故多帧拆分不是编译尺寸被迫，而是时间轴异步控制流 + 编排便利。**
2. **编译成本由「被引用 class」主导**。ffdec 实测：577 个 class 的 DoInitAction（2.67MB） vs 46 个帧脚本 DoAction（0.74MB）≈ **3.59:1**。磁盘 ~970 个 `org.flashNight` class 仅 **577** 被嵌入（CS6 只编译被引用 class）。**帧脚本外置化不减总编译量；攻击点是「被编译 class 子集」。**
3. **class 单一归属当前已全局成立（intersection = 0）**。实测：主 SWF（`CRAZYFLASHER7MercenaryEmpire.swf`）嵌入 `org.flashNight.*` class **0 个**（只有 11 个 `mx.*` 视频组件类）；asLoader.swf 嵌入 **577** 个；**两者交集为 0**。→ 用户担心的「存档/服务器 class 双嵌、asLoader 重编无法覆盖主 SWF」**当前不存在**（2026-06 child-only 加固已根治）。`tools/audit-as2-class-embedding.js` 现仅守 SaveManager/ServerManager 两个 marker。
4. **编译链已无条件 close+reopen**。`compile_action.jsfl:29-32` 每次编译都 `closeDocument(doc,false)+openDocument(docUri)` 从磁盘整体重读（含 symbol CDATA）。→ C2 的 reopen 痛**不是缓存问题**，是「模块增删要去编辑 symbol XML 文本」的维护面 / 认知负担。装备帧（`asLoader.xml:842-843`）早冻结成 2 行 `#include`，git 实锤装备迭代只改 `装备函数列表.as`、从不碰 symbol XML。
5. **FQN 是本仓已验证的去污染手段**。`通信_fs_bootstrap.as` 注释明记同帧 `#include` 拼接致「叶名称冲突」，已全程改用全限定名 `org.flashNight.neur.Server.BootstrapHandshake` 绕开。仓内另有 ~95 处 FQN 生产先例。
6. **`import` 不能出现在函数 / 方法体内**。→ AS2 class 的方法**无法 `#include` 含 `import` 的帧脚本文件**。这是 C2「单脚本」落地形态的硬约束（详 §4）。
7. **boot 是跨 SWF、强顺序、含异步副作用的交织**：
   - f4 用 **`_root.gotoAndStop("boot_check")`** 驱动**主 SWF** playhead（`boot_check` 非 asLoader 标签）——跨 SWF 控制权移交。
   - 同步 `#include` 帧在 include-time **就执行有序副作用**（`ShellSystem.initialize()` / `EventBus.subscribe` / `MissileConfig.loadConfigs(...)` 异步 fire），**不是纯函数定义**。
   - f5/f6/f75 用 `this.stop()` + 回调 `play()` 硬门控；f7 `ParseTaskData` 消费 f5 产 `rawTaskData` + f6 产 `rawTextData`；f11/f25/f27 内部 `gotoAndPlay(preload/load/kill)` 跳空帧（疑似进度条节拍）；f91 `_root.play()+this.removeMovieClip()` 自卸载。

## 2. 决策结论（四分叉拍板）

| 分叉 | 决策 | 备注 |
|---|---|---|
| **C2** | **B 方案：全量 BootSequencer 单脚本**（含异步重写） | 用户两轮明确选最大化纯洁度：symbol 内真正只留 1 个帧脚本，异步序列 + 握手 + 自卸载全部内化进 onEnterFrame 状态机 |
| **C3** | **纪律 + lint 报告态** | BOM 缺失 / import 冲突 → 硬阻断；FQN 一致性 / 保留字 → 先警告观察 |
| **C4** | **只守不切** | 不拆 framework/gameplay 双 SWF（intersection 已 0、29 条上行 arki 边 + load-order 风险不值）。扩 audit 为全局单一归属门 + 死类剔除 |
| **class 迁移** | **不绑定本方案** | 141 个 `_root` 函数文件迁真 class 留作独立长期主线 |

## 3. 目标架构

### 3.1 单帧驱动模型（C2）

asLoader symbol 的时间轴塌缩为**单帧单脚本**。该帧脚本结构：

```as
// === Part A: 唯一的、策划过的通配 import 联合头（lint 查叶名冲突，冲突处子文件改 FQN） ===
import org.flashNight.xxx.*;   // 所有 stage 需要的包之并集（去重 + 碰撞检查）

// === Part B: #include 所有「同步代码定义」文件，但每个 group 体已包成 staged init 函数 ===
//   关键：子文件剥掉自带 import（靠 Part A 联合头解析简单名）；
//        body 包进 _root.__boot.X = function(){ ...原帧体... }，定义≠执行。
#include "../asLoaderManifest/stage_引擎.as"
#include "../asLoaderManifest/stage_通信.as"
#include "../asLoaderManifest/stage_单位函数.as"
... （引擎/通信/单位函数/装备/功能/关卡/战斗/UI/视觉/系统分区/最终化1-3 全部）

// === Part C: 把「有序 stage + 异步 await + 自卸载」交给 BootSequencer 类 ===
org.flashNight.boot.BootSequencer.run(this, _root.__boot);
```

要点：
- **Part B 把「定义」与「执行」解耦**：原帧 include-time 的有序副作用（initialize/subscribe/loadConfigs）改由 BootSequencer 在对应 stage 调用，**精确复刻原帧执行顺序**（§1 事实 7）。
- **联合 import 头放帧顶**：编译单元（帧）作用域内简单名处处可解析，含嵌套函数体内——故「body 包进函数」与「import 在帧顶」不冲突（绕开 §1 事实 6 的方法体禁 import）。
- BootSequencer 是普通 class（住 `类定义/`，编进 asLoader.swf），**不算「symbol 内的脚本」**——满足 C2「影片剪辑内只一个脚本」。

### 3.2 BootSequencer 类（异步状态机）

`org.flashNight.boot.BootSequencer`：
- **由 `_root` 挂载的 tick clip 的 `onEnterFrame` 驱动**（照搬 `DataQueryService.whenAvailable` 的存活模式：等待态在 class 方法 activation 内，asLoader 自卸载后回调仍存活——避开帧本地闭包随 MC 销毁的坑）。
- 按**精确顺序**执行 stage 序列：`引擎()→通信()→awaitHandshake(→_root.gotoAndStop("boot_check"))→awaitTaskData()→awaitTaskText()→parseTask()→guideLoad()→逻辑系统分区初始化()→兼容()→最终化1/2/3()→单位函数()→装备/功能/关卡/战斗/UI/视觉()→[f48-75 各数据源 load/await]→awaitCrafting()→selfUnload()`。
- 复刻三类异步原语：硬门控（stop+回调）改为「状态索引不前进直到 done 标志」；fire-and-continue 改为「fire 后立即进下一 stage」；onEnterFrame 轮询（f4 握手）改为 sequencer 自身的 tick 分支。
- **保留跨 SWF 副作用**：`_root.gotoAndStop("boot_check")`、`_root.play()`、进度文本 `打印加载内容(...)` 的调用时点逐一对齐。

### 3.3 C3 import 治理

- **联合头碰撞检查器**（新 lint，`tools/`）：扫 Part A 联合头里跨包的简单名冲突（两个包都有 `Bar`）→ 报告，冲突处子文件改 FQN。
- **lint 项**：(a) 帧脚本里残留的具体 import；(b) 同单元跨文件简单名冲突；(c) **手工 FQN 转写与原 import 解析一致性**（防把名解析风险换成转写风险）；(d) BOM 缺失（硬阻断）；(e) Flash4 保留字（eq/ne/lt/add/or… 守新命名）。
- 报告态起步；BOM/冲突硬阻断，其余先警告。

### 3.4 C4 class 单一归属守门（只守不切）

- **扩 `audit-as2-class-embedding.js`**：从 2 个 marker → 全局断言「主 SWF 嵌入 `org.flashNight.*` = 0」+「每个 class 恰在一个 SWF」。一旦主时间轴帧脚本误引用游戏 class（致其嵌进主 SWF），门即变红。
- **死类剔除**：~970 disk vs 577 embedded → ~163 非测试未引用类（含 160 Test）。审计确认无引用后裁出编译面——攻击 §1 事实 2 的主导项，零运行时风险。
- **不拆 SWF**：所有游戏 class 留 asLoader 单一归属。compile-speed 拆分（framework/gameplay）因 29 条上行 arki 边（`GlobalInitializer` frame1 即引用 `SceneManager`/`BitmapEffectRenderer`）+ load-order 未证风险，**本轮不做**。

## 4. C1 行为同一性验证策略（双轨——关键）

重构含两类改动，**验证手段不同**：

- **轨道一·字节金标准（适用于「源码级平移」）**：纯把 CDATA 多行 `#include` 逐字节搬进外置 manifest（顺序/相对路径拼写保持）= 编译期内联后字节等价。用 `ffdec -dumpSWF` diff 有序 DoAction(tag12)+DoInitAction(tag59) 标签边界与逐 tag 字节数，零 diff = 行为不变。**agent 可秒级自动跑。**
- **轨道二·trace 等价（适用于 BootSequencer 异步重写）**：BootSequencer 改变了 bytecode 结构，**字节门不适用**。改为**行为 trace 等价**：给「当前 boot」插桩，发出 `(stage, tick序号, 关键 _root 状态突变, 异步 fire/resolve 事件)` 的有序 golden trace；BootSequencer 发同 schema trace；diff 序列。+ 低压平板真机跑完整 boot（验握手→boot_check 主 SWF 恢复、f5/f6→f7 载入顺序、进度条节拍观感、f91 自卸载）。**这是全工程最高 C1 风险项，人类必须真机把关。**

- **BOM 门（双轨共用，必新增）**：每个 `#include`/生成的 .as 断言 `head -c3 == EF BB BF` + ffdec 导出断言每帧 DoAction 非零字节——堵现有 `compile_test.ps1` 抓不到的「丢 BOM → DoAction 0 字节 → compiler 仍报 0 错误」静默坑（本方案大量外置/生成 .as 会**放大**此坑）。

## 6. 残余风险与未决项

- **R1（最高）·异步 trace 等价的充分性**：trace schema 若漏采某维度（如某 _root 字段写时序、进度条帧节拍），可能放过语义漂移。缓解：P0 trace 插桩要尽量全，且 P4/P5 必真机。
- **R2·联合 import 头碰撞面**：全量 169 文件的 import 并集，简单名碰撞概率上升。缓解：lint 碰撞检查 + 冲突处 FQN；必要时把 Part A 拆成 2-3 个分单元（仍单帧多 `#include`）。
- **R3·常量池上限**：全量同步代码并入单帧若逼近 u16(65535) 常量池。缓解：P3/P5 试编 + ffdec 核对；逼近则回退多 staged 帧（仍外置、仍解决 C2 痛）。
- **R4·跨 SWF 时序 + 主 FLA 耦合（2026-06-16 已映射，降级）**：**主文件也是 XFL（`CRAZYFLASHER7MercenaryEmpire/DOMDocument.xml`），boot_check 契约 source-可见**。8-agent workflow 已产权威契约 → [asLoader-BootSequencer-构建标准-2026-06-16.md](asLoader-BootSequencer-构建标准-2026-06-16.md)。结论：恢复 = MAIN f33 `_root.asLoader.play()`（**唯一一行**，DOMDocument.xml:1271）；CHECK3 修正「单帧 play() 不回 frame0、靠帧首 stop+幂等 guard 防重跑」；发现第二耦合点（实例 keyframe 寿命 MAIN f1-33，:1684/:1696）。**全量异步-B 的主 FLA 成本 = 3 处改动**（A 必须:f33 resume 信号 / B 必须:实例 keyframe 或 attachMovie / C 建议:删 f62-64 死代码），主 SWF 仅 11 mx 类重编快。**握手裁决**：asLoader f4 唯一驱动，MAIN f62-64 不可达死代码。仍需真机验 7 项（C2-β/socket 超时/shim 缺失/存档三分支/最终化2/生命周期/loop 不重载）。
