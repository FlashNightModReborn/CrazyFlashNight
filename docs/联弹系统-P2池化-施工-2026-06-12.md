# 联弹系统 P2 池化施工记录 + P3 去MC化审计（2026-06-12）

> 主线背景与 P0/P1（配置治理 + 纵向补弹速率）见会话记忆 `chain-bullet-modernization.md`。
> 本文记录 P2（单元体池化）施工与 P3（去影片剪辑化）审计结论，供验收与后续施工使用。

## 一、P2 施工内容

### 1.1 架构

- **新类** [ChainUnitManager.as](../scripts/类定义/org/flashNight/arki/bullet/BulletComponent/Chain/ChainUnitManager.as)：
  - 共享层 `gameworld.子弹区域.联弹单元体层`（深度 1048000，懒建；gameworld 重建后自动重建并重置组注册表）
  - 按 linkage（`单元体-{子弹种类}`）池化：自由表 + `_visible` 切换，无 attachMovie/removeMovieClip 周转
  - 组注册表 + 单一 `layer.onEnterFrame` 统一 tick（替代旧实现每发联弹一个 onEnterFrame 闭包）
  - 组删除/单元体删除均为 swap-with-last + pop（O(1)）
  - teardown 双保险：消失帧脚本显式 `removeGroupByBullet` + tick 检测 `area/bullet._parent == undefined` 兜底
- **帧脚本重写** [战斗系统_fs_联弹管理.as](../scripts/逻辑/战斗系统/战斗系统_fs_联弹管理.as)：
  - 五类型（横向/横向拖尾/纵向/滑翔/爆炸）全部迁移为 `初始化(clip)`（建组+生成单元体）+ `X更新(group)`（统一 tick 分发）
  - **模拟保持在子弹本地坐标系**，数学公式与旧实现逐式等价；单元体为纯数据 `{mc,x,y,rot,...}`
  - **渲染**：本地坐标经子弹仿射矩阵（`_x/_y/_rotation/_xscale/_yscale`，支持镜像兜底分支）映射到共享层；共享层与子弹同为 子弹区域 直接子级，无需 localToGlobal
  - **area 子剪辑仍是碰撞代理**：包围盒公式不变 → FLAG_CHAIN 碰撞管线零改动
  - **FLA 零改动**：`X联弹初始化(this=area)` / `联弹消失(this=子弹)` / `爆炸联弹消失(this=子弹)` 入口签名不变
  - 旧 `_root.创建单元体` / `_root.回收单元体` 退役（外部零引用已验证）

### 1.2 有意行为差异（验收口径）

1. 衰竭回收 `splice(j--,1)` 的旧实现每次删除会多跳过一个单元体一帧更新（疑似笔误），新实现 swap-pop 不跳帧，视觉更平滑
2. 消弹直删等不经消失帧的销毁路径，单元体由 tick 兜底回收，**至多 1 帧视觉残留**
3. 单元体渲染层级固定在 子弹区域 顶层（旧实现跟随各自子弹深度）
4. `爆炸联弹预处理` 首次被正确注册——旧文件中它在 `_root.联弹系统 = {}` 重置**之前**赋值，实际从未注册（爆炸联弹此前预处理静默未执行）
5. `爆炸联弹消失` 的逐单元体爆炸定位修正为真实单元体位置（旧代码对单元体自身调 localToGlobal 又传入其自身坐标，偏移叠加两次）
6. 统一 tick 与子弹自身 onEnterFrame 的帧内相位可能与旧实现存在亚帧级次序差（包围盒为本地坐标计算，不受影响）

## 二、P3 去MC化审计结论

### 2.1 MC 依赖面（FLAG_CHAIN 路径硬绑定，三处）

| # | 依赖 | 位置 | 含义 |
|---|------|------|------|
| 1 | `bullet.onEnterFrame = BulletQueueProcessor.createNormalPreCheck(target)` | BulletLifecycle.bindFrameHandler | 帧泵入口：子弹靠自身 MC 帧事件进入队列处理。纯对象无帧事件 → 需在 BulletQueueProcessor 增设对象子弹泵列表（对标 `_rayBullets`，但多帧存活） |
| 2 | `bullet.gotoAndPlay("消失")` | BulletQueueProcessor 单出口收尾（VANISH/击中地图） | 击杀路径依赖 FLA 帧标签（消失帧脚本承载死亡行为）。可仿 `bindSafeRemove` 在对象上安装 `gotoAndPlay` 方法垫片转发到 `_root.联弹系统.联弹消失` |
| 3 | `factory.createFromBullet(target, target.area)` + 旋转联弹多边形惰性创建 + CoverageAABB | NormalBulletLifecycle.bindCollider / ChainDetector / BulletQueueProcessor | AABB/Coverage/Polygon 碰撞器全部以 area MC 为输入。需新增 `updateFromChainData`（从组本地极值+子弹矩阵直接算 OBB/AABB 点集）并在 ColliderFactory 三处接线 |

辅助事实：`bindSafeRemove` 已有透明子弹（纯 Object）双模先例；`shouldDestroy`/`checkMapCollision`/movement 字段均可在纯对象上工作；DepthManager 不介入子弹。

### 2.2 施工方案（押后至 P2 真机验收通过后执行）

1. BulletFactory 联弹分支：`_root.对象浅拷贝(Obj)` 创建对象子弹（保留 xmov/ymov/movement 绑定），注册进新泵列表
2. BulletQueueProcessor：新增 `_chainObjectBullets` 多帧泵（进入主扫描的入口与 MC 子弹同构），对象子弹安装 `gotoAndPlay`/`removeMovieClip` 垫片
3. 碰撞器：`AABBCollider.updateFromChainData(group)` + Polygon 同名路径 + ChainDetector 工厂选择改造
4. 帧脚本：初始化入口从 FLA onClipEvent 迁到 BulletFactory 分支内直接调用（按 `子弹种类` 前缀查模板注册表），9 个 FLA 元件 + `_root.联弹系统` 帧标签语义退役
5. 每步 compile 门控 + 真机回归（消弹三模式/穿透/击中地图/爆炸联弹全路径）

**降级决策依据**：①~③ 均处 BulletQueueProcessor 热循环及其契约层，盲改（无真机验证窗口）与 P2 未验收叠加的风险不对称；P2 已消除主要性能开销（attachMovie 周转 + 每弹闭包），薄壳 MC 的边际成本 = 每发一个两帧空壳 + area 子件。
