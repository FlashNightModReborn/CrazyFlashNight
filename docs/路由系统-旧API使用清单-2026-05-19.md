## 8. 第三阶段开工要点（从盘点推导）

1. **`_root.路由基础.*` 进入冻结期** — 0 外部调用，文件保留但不再扩，所有新功能直走 RoutingIntent / RoutingLifecycle class。
2. **搓招声明（兵器/空手）模式化**可考虑后续从分散 .as 抽出 CombatRouteRegistry（35 + 14 处 `<招式名>` 字符串硬编码），但与第三阶段夹具无直接关系。
3. **跨容器跳转（空手 3 处 XML）** 是黑箱夹具的高优先级覆盖目标 —— 它独立于 attachMovie 主路径，且包含 `attachMovie + removeMovieClip` 双向链。
4. **战技/技能容器交叉（升龙拳等 4 个技能容器但调战技动画完毕）** 是夹具契约的边界 case。
5. **`_root.空中控制器` 野生调用面（业务 .as 8 + XML 8）** 意味着夹具必须支持替换 `_root.空中控制器` 实例本身，而不仅是 RoutingRuntime 注入 —— **建议第三阶段开工时把 `_root.空中控制器` 注入接口提升到与 RoutingRuntime 同级**。
6. **状态改变 XML 调用 860 处** 是回归测试黄金量级 —— 第三阶段端到端套件应能在 testloader 内模拟其中一个代表性 mc 的全部状态切换，并对照 producer-set → StateTransition.apply → executeStateTransitionJob → callback 链做断言。
7. **AIEnvironment.routeSkill** 这种"子系统 adapter"是合规模板 —— 未来其他子系统接 `_root.X` 时可仿照（statically wrapped + 测试 mockable）。
