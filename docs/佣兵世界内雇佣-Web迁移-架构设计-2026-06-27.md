## 1. 背景与目标

游戏世界里碰到**可雇佣兵 / 可招募战宠 NPC**（`MercSpawner` 生成、`受雇欲望=5`），右键 NPC → 选项菜单（Symbol 1770）→「雇佣」→ 弹出确认框 **Symbol 2035（`_root.佣兵确认面板`）** 付费雇佣。这是一条**仍然活着、纯 AS2/FLA**的世界内雇佣路径，和已 web 化的战队页管理雇佣**双轨并存、逻辑重复**。

**目标**：把世界内雇佣（佣兵 + 战宠双用）迁进 web 战队页，复刻副本任务那套"web 确认 / AS2 权威写"；顺手清掉随之而来的一簇佣兵 orphan 死码。

**非目标**：战队页本身的管理/浏览雇佣（`mercHire`/`petAdopt`）已 web 化，不动；本设计只迁**世界内单目标雇佣**。

- 据 `_root.gameworld[npcId].宠物数据 != null` 判 `kind = "pet" | "merc"`

### 3.3 `hireConfirm`（web→AS2，权威写）

## 4. AS2 权威写入（handleWorldHire / handleWorldAdopt）

复刻 Symbol 2035 的 `结算`+`雇佣佣兵`/`雇佣宠物`+`虚拟币支付`，**权威读 `_root.gameworld[_pendingHireNpc]`**（web 不可信）：

```
handleWorldHire(params):
  npc = _root.gameworld[_root._pendingHireNpc]
  if npc == undefined or npc.佣兵数据 == undefined: resp(false, "npc_gone"); return
  merc = npc.佣兵数据
  // 门控（复刻 结算 + 雇佣佣兵）：槽位 / 低等级限高级 / 金钱
  targetSlot = 首个空槽[0,佣兵个数限制)；满→ resp(false,"slots_full")
  if 等级+5 < merc[0] and 等级<20 and !isEasy → resp(false,"level_gap"); return
  if 金钱 < 金币价格 → resp(false,"insufficient_gold"); return
  // 扣费 + 写入（复用 handleHire 已抽逻辑：价格/扣款/槽位/同伴数）
  扣 金钱；买佣兵；同伴数据[slot]=merc；佣兵是否出战信息[slot]=1；同伴数=countCompanions()
  可雇佣兵/隐藏的可雇佣兵 splice（是否杂交==false）
  _root.存档系统.dirtyMark = true        // ← 副本 P1b 同款纪律
  成就记账（佣兵雇佣次数/花费金币）
  // 世界 spawn + 删 NPC（AS2 独占，web 做不到）
  spawnCompanionAt(npc._x, npc._y, merc, slot)   // 内联 attachMovie("主角-男",...是否为佣兵:true)
  removeMovieClip(npc)
  _root._pendingHireNpc = undefined
  resp(true, hired:true)                  // web 收 hired → 关面板交还 Flash
```
`handleWorldAdopt` 同构（宠物：栏位扫描 / `宠物信息[格子]` / `加载宠物` / `removeMovieClip`）。

**⚠ 定价/出战不可共享（实测修正）**：世界内雇佣与 roster 雇佣**经济不同**，`handleWorldHire` 须**忠实复刻 Symbol 2035**，非复用 `handleHire`：
| | roster `handleHire` | 世界内 `handleWorldHire`（复刻 Symbol 2035 `刷新佣兵数据`:39-71 / `雇佣佣兵`:171-244） |
|---|---|---|
| 金币价 | `floor(merc[18]*1.5)` | `floor(merc[18] * 口才折扣)`（`口才折扣 = 1 - 主角被动技能.口才.等级*0.02`，启用时） |
| K点 | 等级≥50 时 `(lv-50)*100` | **0**（世界内雇佣无 K点） |
| 出战默认 | `佣兵是否出战信息=0`（休息） | **`=1`（出战）** |
| 额外 | 价格倍率 `merc[19].价格倍率` | `买佣兵(merc[2], floor(金币价*0.8))` + `可雇佣兵`/`隐藏的可雇佣兵` splice + spawn + 删 NPC |
**可共享的仅**：`countCompanions()` 槽位扫描 + `存档系统.dirtyMark` 纪律 + 成就记账惯例。其余逐字复刻 Symbol 2035（含 `结算`:129-169 门控顺序 + `虚拟币支付`:270-295 分流）。

**安全姿态（= 副本 dungeonEnter）**：金额/数据/限制一律 AS2 读权威；web 仅 confirm；`_pendingHireNpc` 由 NPC 交互写、hireConfirm 用后清，web 无法雇任意 NPC。

## 5. Web 视图（候选置顶进 roster — 内聚式）

> **设计修订（2026-06-27，用户真机反馈后）**：原"单目标确认浮层"改为**候选置顶进战队 roster**。内聚式更优：玩家可与现役队员**同屏比较**、满员可**先解雇/删宠腾位再雇**、视觉用**真·战队卡**一致。AS2/C# 协议不变，仅 web 呈现层改。

team-panel.js `onOpen`：`initData.view==="hire"` → 把 `detail` 作 `hireCandidate` 传给 kind 匹配的控制器（`candidateFor` 只投匹配 tab，切走再切回不丢），**不再独立浮层**：
- 控制器 `renderListPage`/`renderPetGrid` 在现役卡**网格顶部置顶一张候选卡**（merc 复用 `buildMercCard('candidate')` 真·战队卡；pet 用 `.pet-card` 视觉的 `buildPetCandidateCard`），带「待雇佣/待招募」青绿高亮 + 价格 + 雇佣/招募 CTA。
- **门控按实时 `_snapshot` 复算**（slots/gold）+ AS2 下发的 `levelGap`（merc）：满员/钱不足/等级过高 → 按钮禁用直写原因；**解雇/删宠 → `requestSnapshot` 重渲染 → 候选自动解禁**（这正是内聚式的价值）。
- 「雇佣/招募」→ `world_hire`(mercs)/`world_adopt`(pets)（`_busy`/`beginOp` 锁防连点）；回 `hired:true` → `requestClose()` 关面板（场景里 NPC 已被 AS2 删、单位已生成/加载）。
- 门控拒绝（slots_full/level_gap/insufficient_gold/npc_gone）→ toast，保留面板。「取消」= 面板自身关闭钮，不写。

## 8. 风险 / 陷阱

- **⚠ web 面板 = 暂停 + NPC 离场韧性（2026-06-27 实测修正，双层防护）**：可雇 NPC **有寿命会自行 despawn**（`MecenaryBehavior.think` `aliveFrames > ALIVE_MAX_TIME → 删除可雇用单位`），而原先**只有 kshop 暂停游戏**、team 面板不暂停 → 玩家在面板里比较/解雇期间 NPC 会离场。两层修复：
  - **① 中央暂停（用户意图：开 web 面板 = 暂停，此时玩家看不到 AS2 画面，游戏不该在背后跑）**：C# `LauncherCommandRouter.OpenPanel` 任意面板打开 → `webPanelPause`、`WebOverlayForm` `case "close"` → `webPanelUnpause`（覆盖 panelHost+fallback 两条开面板路 + 所有 web 关闭）；AS2 `UI管理.as` 注册幂等 lease（`_root._webPanelPauseLease` 只持一个，多次 open 含 returnTo 不叠加；与 kshop 的 "shop" lease 叠加安全——两个都释放才真正解暂停）。**所有 web 面板**自此开即暂停、关即恢复。
  - **② NPC 数据 stash（defense-in-depth，复刻旧 Symbol 2035）**：`handleOpenWebHire` 改 stash 数据快照 `_root._pendingHire = {npcId, merc, pet, 雇佣价格}`（不读 live `gameworld[npcId]`，survive despawn）；`handleWorldHire`/`handleWorldAdopt` 用 stash 数据 + spawn 位 = NPC 还在→其位 + removeMovieClip、已离场→玩家位、跳过 remove（复刻 `雇佣佣兵:199-208` 兜底）。即便暂停某天漏失，NPC 走了仍能雇。`npc_gone` 仅在 stash 本身缺失时。安全姿态不变（stash 数据、不信 web）。
  - 验证：asLoader publish 0 错 + `run_tests.ps1` 628/0 + `build.ps1` 绿（Core.dll 部署）。
- **`买佣兵` / `加载宠物` / `attachMovie("主角-男",...)` 是 AS2 权威副作用**，逐字复刻 Symbol 2035，勿改经济/spawn 语义（同副本"内联复刻、显式 _root 赋值"纪律）。
- **AVM1 静默**：world spawn 的 `attachMovie` 参数对象字段名（脸型/发型/各装备槽/性别/是否为佣兵:true）须与 Symbol 2035:234 逐一对齐，漏字段=单位渲染缺件。
- **双 task 通道**：merc 走 `"mercs"`、pet 走 `"pets"`，勿混（信封 panel 决定路由到 MercTask/PetTask）。
- **things0 SWF**：死调用清理要重编 things0（独立 SWF），列入退役阶段清单，别漏。
- **dirtyMark**：世界内雇佣扣 金钱/虚拟币 + 写 同伴数据/宠物信息，全 save-relevant，**必 `存档系统.dirtyMark=true`**（副本 P1b 教训）。
