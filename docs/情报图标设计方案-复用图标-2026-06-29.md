# 情报图标设计方案：复用图标独占化

日期：2026-06-29

## 范围

本表只覆盖 `data/items/收集品_情报.xml` 中满足以下条件的情报物品：

- `name != icon`
- 同一个 `icon` 在情报物品内被使用超过 1 次

当前目标共 41 条，涉及 6 个旧复用 icon：`烹饪指南`、`报纸1`、`废城资料`、`黑铁会档案图标`、`报纸2`、`制作图纸默认`。所有目标均存在对应 `data/intelligence/<name>.txt`；其中 3 条制作图纸正文当前为空页，仅能基于名称与 XML description 设计。

## 共用风格

沿用现有 launcher item icon 语言：透明背景或黑底可读、单主体居中、粗黑描边、高对比大色块、少量主题色、弱明暗、32x32 优先。禁止把情报正文画成复杂插画、场景、人物肖像、文字海报或 UI 卡片。每个方案只保留一个主轮廓和一个辅助语义点。

Kimi CLI 已按 5 批完成反方复核。下表为吸收 Kimi 意见后的人工验收版；`Kimi 审阅` 列记录该条是通过、已瘦身修订，还是需要人工确认后再生图。

## 游戏资产参照挖掘

结论：值得做，而且应作为生图前的固定步骤。现有 `launcher/web/icons/manifest.json`、`data/crafting/烹饪.json`、关卡 XML、launcher 人物/势力背景图已经能为多数情报提供项目内视觉锚点。参照图只用于约束色块、轮廓、材质和游戏内语境，不直接拼贴或复刻成品图标；情报图标仍以“文件/纸张/档案/图纸”为主轮廓，资产元素退为 1 个辅助语义点。

分级：`强` 表示有同物品、同关卡掉落或同系列图标可直接做视觉参考；`中` 表示有同势力、同关卡或同材料族参考，但没有完全对应主体；`弱` 表示只能用通用风格或剧情语义；`阻断` 表示数据仍需确认后再进生图。涉及外部梗时只保留语义，不把外部 IP 原图作为参考输入。
