## 一、视觉重设计（战术军事风）

对齐任务面板的视觉语言，全部用纯 CSS 实现（不依赖新美术资源）：

- **固定画布 1024×576 + scale-shell**：`updateFitScale / bindScaleWatcher` 照搬 task-panel.js，
  `--pet-scale = min(w/1024, h/576)`，几何居中后整体缩放。
- **配色**：战术橙 `#ef9442` 主调（取代旧青蓝 `#00bcd4`）；深哑光底 + 顶部橙色辉光 + 网点纹理。
- **战术角标框**：纯 CSS（`::after` 8 段 linear-gradient 画 4 角 L 形角标），复用 `.pet-bracket`；
  外框 `.pet-panel::after` 画四角。
- **切角按钮**：`clip-path: polygon(...)` 倒角，区分主行动（橙实心）/ 次级（描边）/ 危险（红）。
- **半调网点纹理**：复用 task_panel.css 的 `radial-gradient(...) / 5px 5px`，用于底部选择栏与弹窗。
- **思源宋体 CN Heavy** 标题（全局 @font-face，已有；headless 预览回退 serif）。
- **动效**：卡片入场 stagger、hover 上浮 + 扫光、进度条填充、页面淡入/滑入、角标/数字脉冲、MAX 经验条流光。

## 二、布局（吸收 AI mockup 思路）

仍为三页（列表 / 领养商店 / 进阶），但列表页重构为 mockup 形态：

- **Header**：三角标 + 「战宠管理」+ 金币/K点资源 + 关闭
- **Toolbar**：出战 N/M、宠物栏 N/M 状态 + **排序下拉** + **筛选下拉** + 领养宠物（橙实心）
- **卡片网格**：4 列；卡片含头像 / 名称+Lv / 徽章（出战中·MAX·体力耗尽）/ 体力·经验条 / 出战·恢复迷你按钮；
  出战态左侧绿条 + 选中态橙框辉光；末尾「空闲栏位 / 开格子」卡片
- **底部「当前选择」常驻栏**：选中宠头像 + 名称 + 状态 chip + 体力/经验条 + 出战·恢复·培养按钮

商店页 / 进阶页同步换战术皮（分类 tab、领养卡片、属性区、进阶方案列表的开关/循环 chip/锁定四态）。

## 三、体验增强清单

**P0 手感**
- 按钮 **pending 态**：所有异步操作（出战/恢复/强化/删除/进阶/领养/开格子）触发后按钮转 spinner，
  `beginOp/endOp` 统一管理 + 面板整体忙时拦截，消除「点了没反应重复点」
- **Toast 三色队列**：成功（绿✓）/ 失败（红✕）/ 警告（黄!），最多 3 条堆叠，不再单条互盖
- **进度条动画**：`pet-bar-grow` 入场 + `transition:width` 变化过渡

**P1 流畅**
- **卡片局部更新**：出战/恢复后只替换单张卡片节点（`refreshCard`），不整页重排，无闪烁/滚动丢失/卡片跳走

- **页面切换过渡**：淡入 + 方向滑入（前进/返回方向相反）
- **loading 骨架**：首次 snapshot 前显示 8 张骨架卡

**P2 精致**
- **排序**：出战优先 / 等级↓ / 等级↑ / 体力↑ / 名称；**筛选**：全部 / 仅出战 / 仅休息 / 体力不足

- **删除确认弹窗重做**：动画进入 + 宠物名加粗 + 返还灵石独立强调块

## 六、栏位交互模型调整（2026-06-06 追加）

用户反馈后调整领养/栏位的交互直觉（数据语义已核对：`_pets` 仅含非空槽，
`maxSlots = _root.宠物领养限制`=当前容量，空位 = `maxSlots - owned`，
`开宠物格子()` push 空数组令容量+1，领养自动填下一个空槽）：

- **领养宠物 ↔ 开格子 调换位置**：工具栏的「领养宠物」按钮 → 改为「＋ 开格子」（橙实心）；
  扩容入口上移到工具栏。
- **全量显示空位**：网格在已拥有卡片之后，渲染 `maxSlots - owned` 张「领养空位」虚线卡片，
  玩家可直观看到剩余栏位数（随网格滚动）。
- **点空位即领养**：点击任一空位卡片 → 进入领养商店；服务端 `handleAdopt` 自动填入下一个空槽。
- 仅「全部」筛选下显示空位卡片；`开格子` 错误细分（已达最大格子数 / 金币不足）走 warn toast。

文件改动仅 `pet-panel.js`（toolbar 按钮 + renderPetGrid 空位循环 + onExpandSlot 错误文案，
删 appendExpandCard）与 `pet_panel.css`（`.pet-expand-card*` → `.pet-slot-empty`）。

## 七、卡片/选中栏/培养 三层职责差异化（2026-06-06 追加）

用户反馈：底部选中栏重复了卡片的 出战/休息/恢复，只多一个培养，缺乏差异化。调整为三层各司其职：

| 层级 | 职责 | 控件 |
|------|------|------|
| **卡片** | 出战就绪（一眼看 + 快捷战备） | 出战/休息、恢复体力 |
| **选中栏（详情）** | 快捷切换已解锁的反复型进阶 | 淬毒开关、发型循环 chip…（`#pet-sel-quick`）+ 培养入口 |
| **培养页** | 深度养成 + 完整文案 | 强化、删除、一次性进阶、全部进阶方案描述 |

- 选中栏**移除 出战/休息/恢复**，改为 `renderSelbarQuick(pet)`：从 `pet.schemeStatus` 直接派生
  `repeatable && purchased && toggleKind∈{binary,cycle}` 的方案，渲染为开关/循环 chip（不依赖 _petLib）。
- 快捷开关点击 → `onAdvance(scheme, pill, '已更新', _selectedSlot)`。
  **关键修正**：`onAdvance` 原仅认 `_activePetIdx`（培养页目标），列表页选中栏触发时为 -1 会误判；
  改为接受显式 `slotIndex` 参数，省略时回退 `_activePetIdx`。
- 数据依据：AS2 `handleSnapshot` 用 `extractPromotionNames(petDef.Promotion)` 全量构建 `schemeStatus`，
  每项带 `repeatable/purchased/toggleKind/toggleOn/toggleValue/desc`，Web 端可直接消费。
- harness 加 `?sel=狗狗` 驱动（选中含 toggle 的宠物便于截图快捷区）。
- **保留项**：培养页 header 仍含 出战/休息/恢复（聚焦单宠时的便利），未一并移除；若要彻底贯彻
  「培养只管养成」可再砍，待定。
